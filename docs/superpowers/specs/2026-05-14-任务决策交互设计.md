# 任务决策交互设计

## 背景

当前开发闭环在分析阶段会列出「需要确认的问题」，但系统不会读取这些问题，也不会暂停流程。分析完成后会自动进入规划阶段，导致产品经理或分析 Agent 在遇到文件范围、大小限制、是否传给后端和 Agent 这类关键选择时，只能自行假设并继续写计划。

## 目标

当分析阶段发现阻塞规划的产品或技术决策时，系统应根据任务设置决定是暂停等待用户选择，还是采用 Agent 推荐选项自动继续规划。

## 范围

本期实现：

- 任务级交互策略：`需要决策时询问我` 或 `使用推荐选项自动继续`。
- 分析阶段输出结构化决策项。
- 后端解析决策项，并在需要人工决策时进入 `awaiting_decision`。
- 前端任务详情展示待决策问题和选项，允许用户提交选择。
- 用户提交决策后继续规划。
- 自动继续时保存推荐决策产物，保证后续计划可追溯。

本期不实现：

- 真实文件上传、存储和传输。
- 多轮动态追问。
- 在线编辑计划。
- 项目级全局默认策略。

## 状态与数据

Workflow 新增状态：

- `awaiting_decision`：分析阶段已完成，但存在需要用户选择的阻塞决策。

Task 新增字段：

- `interaction_mode`：
  - `ask_user`：有阻塞决策时暂停等待用户。
  - `auto_recommended`：有阻塞决策时使用推荐选项自动继续。

Artifact 新增类型：

- `decision_request`：Agent 提出的结构化决策项。
- `decision_response`：用户选择或系统默认选择的结果。

决策项结构：

```json
{
  "decisions": [
    {
      "id": "file-scope",
      "question": "文件支持范围是否只做图片，还是所有文件？",
      "reason": "该选择影响前端上传控件、后端校验和 Agent 可读取上下文。",
      "blocking": true,
      "recommendedOptionId": "images-only",
      "options": [
        {
          "id": "images-only",
          "label": "仅支持图片",
          "description": "先支持 png、jpg、webp 等图片，降低实现范围。"
        },
        {
          "id": "all-files",
          "label": "支持所有文件",
          "description": "覆盖面更广，但需要更严格的大小、类型和安全策略。"
        }
      ]
    }
  ]
}
```

## 编排流程

1. 启动 workflow 后进入分析阶段。
2. 分析 Agent 输出 Markdown，同时必须附带 JSON 决策块。
3. 后端保存分析产物并解析决策块。
4. 如果没有阻塞决策，继续 planning。
5. 如果存在阻塞决策且任务策略为 `ask_user`，保存 `decision_request`，workflow 进入 `awaiting_decision`。
6. 如果存在阻塞决策且任务策略为 `auto_recommended`，保存 `decision_request` 与 `decision_response`，使用推荐项继续 planning。
7. 用户提交决策后，后端保存 `decision_response`，workflow 回到 `running` 并进入 planning。
8. planning prompt 包含分析、决策请求和决策响应，规划 Agent 必须按已选决策生成计划。

## 前端交互

任务详情中新增「交互策略」选择。默认使用 `需要决策时询问我`。

当 workflow 处于 `awaiting_decision` 时，在开发闭环区域展示「待决策」面板：

- 每个问题显示问题、原因和推荐标记。
- 每个问题使用单选项。
- 用户必须为每个问题选择一个选项。
- 提交按钮文案为「提交决策并继续」。

该面板是工作台式表单，不使用营销式解释文案，不嵌套卡片。

## 错误处理

- 分析输出没有 JSON 决策块时，按没有阻塞决策处理。
- 决策 JSON 结构无效时，保存分析产物并继续 planning，避免流程因模型格式波动中断。
- 用户提交的选项 ID 必须存在于当前最新 `decision_request` 中，否则返回 400。
- 非 `awaiting_decision` 状态提交决策返回 409。
