# 统一技能矩阵页面设计

## 背景

当前 Skills 页面按平台 tab 展示本地 skills：Codex、Claude Code、OpenCode 各自独立列表。用户需要改为“所有 CLI 平台 skills 的去重集合”，并能直接看出每个 skill 安装在哪些平台。页面还需要更现代、更紧凑、更多 icon 化表达，并把在线安装从页面内区域改为弹窗。

## 目标

1. 主页面展示全平台去重后的 skill registry，而不是单平台 tab 列表。
2. 每个去重 skill 行明确展示 Codex、Claude Code、OpenCode 的安装状态。
3. 支持行内“补装缺失平台”入口。
4. skills.sh 在线安装改为弹窗；本地路径导入保留在弹窗的第二个 tab。
5. 页面视觉改为更紧凑、现代、管理台风格：小字号、高信息密度、状态 icon、明确的平台 badge。

## 非目标

1. 不新增远程 CLI 执行验证，例如调用 Codex / Claude / OpenCode 命令检测实际加载结果。
2. 不迁移或重命名用户现有本地 skills。
3. 不做批量删除、批量同步、自动更新等更重管理功能。
4. 不改变现有 copy 默认、symlink 高级选项的产品决策。

## 信息架构

采用已确认的方案 C：Filter Sidebar + Registry。

页面由三块组成：

1. 顶部工具栏
   - 标题：`Skill Matrix`
   - 副标题说明这是 Codex / Claude Code / OpenCode 的统一本地 skill 管理视图。
   - 总数摘要：去重 skill 数、已安装平台计数、异常计数。
   - 操作：刷新、安装。

2. 左侧筛选栏
   - 搜索：按 skill slug、描述、来源过滤。
   - 状态筛选：全部、缺失平台、存在异常。
   - 平台筛选：Codex、Claude Code、OpenCode 已安装 / 未安装。
   - 安装模式筛选：copy、symlink、unknown。

3. 右侧 registry 表格
   - 一行一个去重 skill。
   - 列：Skill、Codex、Claude Code、OpenCode、Health、Actions。
   - 平台列展示 `copy`、`symlink`、`missing`、`invalid`。
   - Health 汇总该 skill 在所有平台上的有效性。
   - Actions 包含详情、删除单平台安装、补装缺失平台。

## 去重规则

第一版按目录名 / skill slug 去重。

理由：

- 当前安装、删除、详情 API 都以目录名为稳定 ID。
- `SKILL.md` frontmatter 的 `name` 可能包含空格或显示名，不适合作为文件系统操作 ID。
- 同名 manifest 可能出现在多个目录，按 manifest name 合并会导致误删或误判。

聚合结果中保留：

- `name`：稳定目录名，作为聚合 ID。
- `displayName`：可选显示名，第一版可直接等于 `name`。
- `description`：优先取有效平台中第一个非空描述。
- `installations`：按平台映射到具体 `PlatformSkill`。
- `missingProviders`：未安装的平台。
- `valid`：所有已安装平台均 valid 时为 true。
- `issues`：聚合所有平台问题，并带平台前缀。

## 后端设计

新增聚合能力，建议增加 endpoint：

```http
GET /api/platform-skills
```

返回：

```ts
interface PlatformSkillAggregate {
  name: string;
  displayName: string;
  description: string | null;
  providers: PlatformSkillProvider[];
  missingProviders: PlatformSkillProvider[];
  installations: Partial<Record<PlatformSkillProvider, PlatformSkill>>;
  installModes: Partial<Record<PlatformSkillProvider, PlatformSkillInstallMode>>;
  valid: boolean;
  issues: Array<{
    provider: PlatformSkillProvider;
    message: string;
  }>;
  lastModifiedAt: number | null;
}
```

服务层实现：

1. 并行调用 `listPlatformSkills(provider)`。
2. 以 `skill.name` 作为 Map key 聚合。
3. 平台顺序固定为 `codex -> claudecode -> opencode`。
4. `description` 选择第一个非空描述。
5. `lastModifiedAt` 取所有安装项最大值。
6. 排序默认按 `name.localeCompare`，异常项可通过前端筛选展示。

现有单平台 API 保留，供详情、删除和兼容旧调用使用。

## 前端设计

`SkillsPage` 从单文件膨胀风险较高，建议拆成局部组件：

- `SkillMatrixPage` 或继续导出 `SkillsPage`，内部拆分。
- `SkillMatrixToolbar`
- `SkillMatrixFilters`
- `SkillMatrixTable`
- `SkillPlatformCell`
- `SkillInstallDialog`
- `SkillDetailPanel`（可保留右侧详情，也可第一版使用表格展开）

状态：

- `searchQuery`
- `statusFilter`: `all | missing | issues`
- `providerFilters`
- `installModeFilter`
- `selectedSkillName`
- `installDialogState`

主数据：

- `useQuery(['platform-skills', 'aggregate'], api.listPlatformSkillAggregates)`
- `useQuery(['platform-skills', 'platforms'], api.listPlatformSkillSummaries)`

页面不再依赖 `activeProvider` 作为主导航。

## 安装弹窗

安装入口：

1. 顶部 `Install` 按钮：打开弹窗，无预选 skill。
2. 行内 `Install missing` 按钮：打开弹窗，预选当前 skill 的缺失平台。
3. 平台单元格中的 missing 状态可作为同一入口。

弹窗结构：

- 标题：Install skill
- Tab 1：skills.sh
  - 搜索框
  - marketplace 结果列表
  - 平台多选，默认 copy
  - 高级区域提供 symlink
- Tab 2：Local path
  - 本地路径输入
  - 平台多选
  - copy / symlink

默认行为：

- 默认安装模式为 copy。
- 行内补装时，targets 默认为该 skill 的 `missingProviders`。
- 如果行内补装没有可靠 source，则仍打开弹窗并提示选择 skills.sh 结果或本地路径。
- 安装成功后刷新 aggregate 和 summary query，并保持当前筛选。

## 删除与详情

删除仍以单平台安装为单位，不做“一键删除所有平台”。

详情区域展示：

- skill slug
- 描述
- 每个平台安装路径、安装模式、版本、来源、状态
- 异常列表

第一版可以用右侧详情面板；如果实现复杂度需要控制，也可在表格行展开显示详情。

## 视觉方向

确认采用紧凑管理台风格：

- 页面背景保持现有应用主题变量。
- 卡片半径不超过 8px。
- 表格行高约 44-52px。
- 使用 lucide icons：
  - refresh：刷新
  - plus / download：安装
  - search：搜索
  - check circle：已安装有效
  - alert triangle：异常
  - link：symlink
  - trash：删除
  - panel / info：详情
- 平台状态使用小 badge 或 icon + 短文字，避免大面积彩色块。
- 信息密度优先，不做营销式 hero。

## 错误与空状态

- 后端请求失败时显示错误框，不显示为“空列表”。
- 无 skills 时显示空状态，并保留安装按钮。
- 某个平台 root 不存在时，在 summary 和平台列中标识为 missing root，不阻塞其他平台展示。
- 单个 skill 无 `SKILL.md` 时仍进入聚合列表，但 Health 为 issue。

## 验证策略

后端：

- 聚合 API 单元测试：多平台同名合并、缺失平台、异常聚合、排序。
- 保留现有安装、删除、安全边界测试。

前端：

- `npm run build -w @openclaw-room/frontend`
- 如项目后续有前端测试 runner，再补筛选和弹窗状态测试。

集成：

- `npm run build`
- dev proxy 下请求 `/api/platform-skills` 返回聚合列表。
- 页面可看到 Codex / Claude Code 已安装数量，OpenCode root 不存在时不导致整页空白。

## 已确认决策

- 使用方案 C：Filter Sidebar + Registry。
- 去重按目录名 / skill slug。
- 本地路径导入保留在安装弹窗第二个 tab。
- 行内提供补装缺失平台入口。
- 默认 copy，symlink 作为高级选项保留。
