# 文件搜索下拉优化设计

- 日期：2026-06-02
- 范围：前端触发器下拉渲染 + 后端搜索返回目录 + 前端 buildFileSuggestions
- 目标：@ 文件搜索下拉支持目录匹配、单行紧凑布局、颜色区分来源、图标区分文件/目录

## 1. 背景

当前 @ 文件搜索下拉：
- 只搜索文件，不搜索目录
- 两行布局（label 上 + description 下）
- 无图标，无颜色区分
- 项目文件与工作区文件视觉上无法区分

## 2. 目标行为

1. 搜索结果同时包含文件和目录
2. 单行紧凑布局：`[icon] [label] ............ [description]`
3. 目录用文件夹图标（amber），项目文件用文档图标（blue），工作区文件用文件图标（emerald）
4. description 右侧显示路径或"项目文件"标签

## 3. 详细设计

### 3.1 后端：searchWorkspaceFiles 返回目录

文件：`packages/backend/src/workspace-files.ts`

- `WorkspaceSearchResult.type` 从 `'file'` 扩展为 `'file' | 'directory'`
- 遍历时对目录名也做匹配，命中则加入结果
- 更新前端类型 `packages/frontend/src/lib/types.ts`

### 3.2 前端：buildFileSuggestions 增加图标

文件：`packages/frontend/src/components/RichMessageComposer.triggers.ts`

- 目录：`icon=<Folder className="h-3.5 w-3.5 text-amber-500" />`，description=相对路径
- 项目文件：`icon=<FileText className="h-3.5 w-3.5 text-blue-500" />`，description="项目文件"
- 工作区文件：`icon=<File className="h-3.5 w-3.5 text-emerald-500" />`，description=相对路径

### 3.3 前端：TriggerPopover 单行布局

文件：`packages/frontend/src/components/prompt-area/trigger-popover.tsx`

当前渲染（两行）：
```
[icon] [label]
       [description]
```

改为单行：
```
[icon] [label ............] [description]
```

- 外层：`flex items-center gap-2`（从 `items-start` 改为 `items-center`）
- label：`truncate font-medium flex-1 min-w-0`
- description：`text-[11px] text-[var(--color-fg-muted)] truncate shrink-0 max-w-[45%] text-right`

### 4. 不改的

- TriggerPopover 容器样式（max-h、阴影、圆角）
- # 任务触发器渲染
- 后端 listWorkspaceDirectory
