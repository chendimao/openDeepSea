# 项目工作区只读文件浏览与聊天引用设计

## 背景

当前项目在新建时已经保存本地目录 `projects.path`，后端也会校验该路径存在且是目录。ACP 运行时也会把项目目录作为当前工作区或可写目录传给 agent。

已有“项目级文件库与群聊引用”设计覆盖的是上传附件库：文件被复制到 `packages/backend/data/uploads/files/<projectId>/`，并通过 `files` 表和 `message_file_refs` 管理。该模型适合上传和复用附件，但不适合直接浏览真实项目源码目录。

本次设计新增“项目工作区只读文件浏览与聊天引用”：用户可以浏览项目目录内的真实文件，并在聊天消息框中快捷引用这些文件。引用不复制文件、不写入项目级附件库，只在消息 metadata 中记录相对路径快照，agent 调度时解析为本地绝对路径用于读取上下文。

## 目标

1. 以 `projects.path` 作为项目工作区根目录，提供只读文件浏览能力。
2. 前端支持在聊天消息框中快速选择工作区文件。
3. 发送消息时可携带工作区文件引用。
4. 后端校验引用路径必须位于项目目录内，防止路径穿越。
5. Agent prompt 中包含被引用工作区文件的文件名、相对路径、大小、MIME 类型和 `localPath`。
6. 与现有项目级文件库并存，不混用数据模型。

## 非目标

1. 不支持写入、删除、重命名、移动、创建目录。
2. 不把工作区文件复制进 `data/uploads/files`。
3. 不把工作区文件写入现有 `files` 表。
4. 不做跨项目文件引用。
5. 不做全文索引、版本管理、Git 状态展示或权限系统。
6. 不暴露裸 `express.static(project.path)` 静态服务。
7. 不默认允许浏览高风险路径，例如 `.git`、`node_modules`、构建产物和环境变量文件。

## 核心概念

### 项目级文件库

已有上传附件库。文件存储在后端数据目录，拥有数据库记录和可下载 URL，适合上传图片、PDF、设计稿、外部资料，并可跨同项目聊天室复用。

### 工作区文件

真实存在于 `projects.path` 下的文件。它的身份是“项目内相对路径”，例如：

```text
packages/frontend/src/components/RichMessageComposer.tsx
```

工作区文件不进入 `files` 表。它的引用随消息保存为 metadata 快照，后端每次调度 agent 时按项目根目录重新解析本地路径。

## 后端设计

### 路径解析

新增共享工具模块，建议命名为 `workspace-files.ts`。

核心函数：

- `resolveWorkspacePath(projectPath, relativePath)`
- `isIgnoredWorkspacePath(relativePath)`
- `statWorkspaceEntry(projectPath, relativePath)`
- `listWorkspaceDirectory(projectPath, relativePath)`
- `buildWorkspaceAttachmentMetadata(project, relativePath)`

路径规则：

1. API 只接受相对路径。
2. 空路径表示项目根目录。
3. 禁止绝对路径。
4. 禁止 `..` 路径穿越。
5. 使用 `resolve(project.path, relativePath)` 后，结果必须等于项目根目录或位于项目根目录之下。
6. 默认不跟随目录 symlink；如文件 symlink 指向项目外，也必须拒绝。
7. 路径分隔符对外统一为 `/`。

### 忽略规则

首版使用内置忽略规则，不引入完整 `.gitignore` 解析。

默认忽略：

- `.git`
- `node_modules`
- `dist`
- `build`
- `.next`
- `.turbo`
- `coverage`
- `.env`
- `.env.*`
- `*.pem`
- `*.key`
- `*.sqlite`
- `*.db`

忽略规则适用于目录列表、文件读取、聊天引用校验和搜索。

### 文件大小限制

首版建议限制：

- 目录列表最多返回 500 项。
- 单个可预览文本文件最大 512 KB。
- 聊天可引用文件最大 2 MB。
- 超过限制的文件仍可在目录中显示，但不能预览或引用，并返回明确原因。

这些限制应集中成常量，便于后续调整。

### API

#### 浏览目录

```http
GET /api/projects/:projectId/workspace/tree?path=<relativePath>
```

返回：

```json
{
  "path": "packages/frontend/src",
  "entries": [
    {
      "name": "components",
      "path": "packages/frontend/src/components",
      "kind": "directory",
      "size": 0,
      "mtime": 1778940000000,
      "ignored": false,
      "selectable": false
    },
    {
      "name": "main.tsx",
      "path": "packages/frontend/src/main.tsx",
      "kind": "file",
      "size": 1234,
      "mtime": 1778940000000,
      "mimeType": "text/typescript-jsx",
      "ignored": false,
      "selectable": true
    }
  ]
}
```

排序规则：

1. 目录在前，文件在后。
2. 同类按名称自然排序。
3. 忽略项默认不返回；如果后续需要调试，可增加 `includeIgnored=1`，但首版不做。

#### 读取文件预览

```http
GET /api/projects/:projectId/workspace/file?path=<relativePath>
```

仅支持文本类文件预览。返回：

```json
{
  "path": "packages/frontend/src/main.tsx",
  "name": "main.tsx",
  "mimeType": "text/typescript-jsx",
  "size": 1234,
  "content": "import React from 'react';\n...",
  "truncated": false
}
```

二进制、大文件、被忽略文件或目录返回 400，并给出错误原因。图片预览首版不通过工作区 API 暴露真实文件 URL；如需要图片查看，应后续设计独立的安全预览接口。

#### 搜索文件

```http
GET /api/projects/:projectId/workspace/search?q=<query>
```

首版只做文件名搜索，不做内容搜索。返回最多 50 个匹配文件，跳过忽略目录和超过引用限制的文件。

### 发送消息扩展

现有消息发送接口继续保留 `fileIds`，新增 `workspaceFilePaths`：

```http
POST /api/rooms/:roomId/messages
Content-Type: application/json

{
  "content": "看看这些文件",
  "mentions": ["room_agent_id"],
  "fileIds": ["uploaded-file-id"],
  "workspaceFilePaths": [
    "packages/frontend/src/components/RichMessageComposer.tsx"
  ]
}
```

multipart 消息也支持 `workspaceFilePaths` 字段，格式为 JSON 字符串数组。

服务端校验：

1. `roomId` 必须存在。
2. room 所属项目必须存在。
3. 每个 `workspaceFilePaths` 必须是项目内相对路径。
4. 路径必须对应普通文件。
5. 文件不能命中忽略规则。
6. 文件大小不能超过聊天引用限制。
7. 同一消息内重复路径去重。
8. 上传附件、项目文件库引用、工作区文件引用的总数受 `MAX_MESSAGE_FILES` 约束，或新增统一的 `MAX_MESSAGE_REFERENCES` 常量。

## Message Metadata

扩展 `MessageAttachmentMetadata`，保持向后兼容。

建议新增字段：

```ts
type MessageAttachmentSource = 'upload' | 'project-file' | 'workspace';

interface MessageAttachmentMetadata {
  id: string;
  fileId?: string;
  source?: MessageAttachmentSource;
  workspacePath?: string;
  name: string;
  mimeType: string;
  size: number;
  url: string;
  isImage: boolean;
  deleted?: boolean;
}
```

工作区文件 metadata 示例：

```json
{
  "attachments": [
    {
      "id": "workspace:packages/frontend/src/main.tsx",
      "source": "workspace",
      "workspacePath": "packages/frontend/src/main.tsx",
      "name": "main.tsx",
      "mimeType": "text/typescript-jsx",
      "size": 1234,
      "url": "workspace://packages/frontend/src/main.tsx",
      "isImage": false
    }
  ]
}
```

`url` 对工作区文件只作为非 HTTP 标识，不用于浏览器直接打开。前端渲染时如果 `source === 'workspace'`，应使用 `workspacePath` 展示路径，不尝试当作下载链接。

## Agent 调度

当前 dispatch 会从附件 URL 解析上传文件本地路径。新增工作区文件解析分支：

1. 解析消息 metadata 中 `source === 'workspace'` 且有 `workspacePath` 的附件。
2. 使用消息所属 room 的项目路径解析 `workspacePath`。
3. 复用工作区路径校验和忽略规则。
4. 成功后在 prompt 中加入 `localPath`。
5. 如果文件已不存在、被忽略或超出项目根目录，prompt 中标记 `localPath=unavailable`。

Agent prompt 文案调整：

```text
消息引用文件：
请结合以下文件回答。如果需要查看内容，请读取对应 localPath。
1. main.tsx | source=workspace | relativePath=packages/frontend/src/main.tsx | mimeType=text/typescript-jsx | size=1234 | localPath=/abs/project/packages/frontend/src/main.tsx
```

图片文件首版不作为 ACP image 参数传入，除非后续确认 ACP adapters 对本地 workspace 图片路径的处理边界。

## 前端设计

### 文件浏览入口

首版以聊天框内选择器为主，不新增独立页面。

在 `RichMessageComposer` 工具栏中保留现有两个入口：

- 本地文件上传
- 项目级文件库选择

新增第三个入口：

- 工作区文件选择

建议图标使用 `FolderTree` 或 `FileCode`，tooltip 文案为“引用工作区文件”。

### `WorkspaceFilePickerDialog`

新增组件职责：

1. 显示当前目录面包屑。
2. 展示目录和文件列表。
3. 支持进入目录、返回上级。
4. 支持文件名搜索。
5. 支持多选可引用文件。
6. 已选文件显示勾选状态。
7. 不可引用文件显示禁用态和原因。
8. 确认后把文件元信息返回给 composer。

列表项展示：

- 文件名
- 相对路径
- 文件大小
- 修改时间
- 文件类型

### Composer attachment 类型

当前 composer attachment 分为本地待上传和项目文件库引用。新增工作区引用：

```ts
type ComposerAttachment =
  | { kind: 'local'; id: string; file: File; previewUrl: string | null }
  | { kind: 'project'; id: string; file: ProjectFile }
  | { kind: 'workspace'; id: string; file: WorkspaceFileEntry };
```

发送时：

- `local` 走 multipart `files`
- `project` 走 `fileIds`
- `workspace` 走 `workspaceFilePaths`

附件栏展示：

- 工作区文件显示文件名和相对路径。
- 不显示图片缩略图。
- 支持移除。

### 消息渲染

消息 metadata 解析需要允许 `source` 和 `workspacePath` 字段。

渲染规则：

1. 上传附件和项目文件库附件继续按 URL 下载或预览。
2. 工作区文件显示为引用卡片，展示文件名和相对路径。
3. 工作区文件不提供直接下载链接。
4. 如果后端后续提供预览接口，可在卡片上打开只读预览弹窗。

## 安全边界

1. 不挂载 `express.static(project.path)`。
2. 不返回真实绝对路径给前端。
3. 只在 agent prompt 内使用 `localPath`，因为 agent 运行在本地并已围绕项目目录工作。
4. API 始终以项目 ID 和相对路径定位文件。
5. 禁止路径穿越、绝对路径和项目外 symlink。
6. 默认忽略敏感文件和重型目录。
7. 工作区文件引用失败时不阻断历史消息展示，只在后续 agent 调度中标记 unavailable。
8. 由于当前后端启用开放 CORS，实现时应同步收紧 CORS 或引入本地会话 token，否则浏览器中的第三方网页可能请求本地 API。

## 错误处理

常见错误：

- `project not found`：项目不存在。
- `workspace path is invalid`：路径非法或越界。
- `workspace path is ignored`：路径命中忽略规则。
- `workspace file not found`：文件不存在。
- `workspace path is not a file`：引用对象不是普通文件。
- `workspace file is too large`：文件超过引用或预览限制。
- `too many files`：总引用数量超过限制。

前端行为：

- 浏览目录失败：在弹窗内展示错误和重试按钮。
- 引用文件失败：toast 提示，并保留编辑器内容。
- 发送时校验失败：沿用当前发送失败提示，不清空 composer。

## 验证策略

### 后端测试

1. 目录浏览只返回项目根内文件。
2. `../` 路径穿越被拒绝。
3. 绝对路径被拒绝。
4. 忽略目录和敏感文件不出现在列表中。
5. symlink 指向项目外时被拒绝。
6. 文件名搜索跳过忽略目录。
7. 发送消息时 `workspaceFilePaths` 写入 metadata。
8. 不存在、过大、被忽略、越界路径不能作为聊天引用。
9. dispatch 能把 workspace metadata 解析为 `localPath`。
10. 文件删除或移动后 dispatch 标记 `localPath=unavailable`。

### 前端验证

1. 工作区文件选择器能打开根目录并进入子目录。
2. 搜索能按文件名筛选。
3. 多选文件后附件栏显示工作区引用。
4. 移除附件后不随消息发送。
5. 发送消息 payload 包含 `workspaceFilePaths`。
6. 历史消息能显示工作区文件引用卡片。

### 最低门禁

实现完成后至少运行：

```bash
npm run build
```

若补充测试，应运行相关后端和前端测试命令。

## 风险与权衡

1. 工作区文件会随本地目录变化而变化；消息保存的是路径快照，不保证未来仍存在。
2. 不返回绝对路径给前端会减少泄露面，但前端不能直接打开本地文件。
3. 不复制文件避免污染附件库，但历史可追溯性弱于上传附件。
4. 忽略规则首版内置，简单可靠；代价是可能和项目 `.gitignore` 不完全一致。
5. 收紧 CORS 或本地 token 是重要安全项，应和本功能同批实现，避免本地文件浏览 API 被跨站调用。

## 已确认决策

1. 只做只读文件浏览。
2. 聊天消息框支持快捷引入工作区文件。
3. 工作区文件不进入项目级文件库。
4. 不开放真实项目目录静态服务。
5. 首版不支持写入、删除、重命名。
6. Agent 通过 prompt 中的 `localPath` 读取被引用文件。
