# ACP 消息分层展示设计

## 背景

当前 ACP 事件已经能进入房间消息的 `trace.events`，但 UI 将部分协议事件按原始粒度直接渲染到 `AgentTimeline`，导致两个问题：

1. `agent_message_chunk` 被映射成 `assistant_message` 后出现在执行过程里，形成大量“助手回复”卡片。
2. `tool_call_update`、`available_commands_update`、raw event 等协议噪声会挤占真正有价值的过程信息。

本设计目标是把 ACP 消息按语义分层展示：用户读正文，开发者看过程，必要时再查协议原始事件。

## 目标

- `agent_message_chunk` 只用于聊天正文流式渲染，不展示在 timeline summary 中。
- thinking / reasoning 进入 timeline，但默认折叠。
- plan 进入 timeline，并以计划步骤列表展示。
- tool call 生命周期按 `toolCallId` 合并成一张卡片。
- diff 从普通事件升级为专门 diff 展示区域。
- `available_commands_update` 不作为普通 timeline 事件展示。
- raw / unknown event 进入“协议调试 / Traffic Monitor”，默认隐藏。
- timeline 事件数量只统计用户可见过程事件，不统计正文 chunk 和默认隐藏调试事件。

## 非目标

- 不改变 ACP provider 连接方式。
- 不改变后端对 raw event 的持久化能力。
- 不删除已有历史消息中的事件数据。
- 不实现完整 IDE 级 diff 编辑器，仅提供可读的 inline diff / diff panel。
- 不把 Traffic Monitor 做成独立开发者工具，只在当前房间消息内提供折叠调试视图。

## 信息架构

ACP 消息在 UI 上分成三层：

### 1. 聊天正文

用于展示 assistant 的最终回复和流式回复。

包含事件：

- `agent_message_chunk`
- 后端标准化后的 `assistant_message`

展示规则：

- 只合并到消息正文。
- 不进入 `AgentTimeline`。
- 不计入 ACP 执行过程事件数量。
- 多个 chunk 按顺序追加，避免在正文中重复已展示前缀。

### 2. 执行过程 Timeline

用于展示用户关心的推理、计划、工具、命令和文件变化。

包含事件：

- `thinking`
- `plan_update`
- `tool_call`
- `tool_result`
- `file_diff`
- `command`
- `command_output`
- `permission_request`

展示规则：

- thinking 默认折叠。
- plan 展示为步骤列表。
- tool call 按 `toolCallId` 合并。
- diff 使用专门的 diff 组件展示。
- command output 可折叠，避免大段 stdout 撑开消息。

### 3. 协议调试 / Traffic Monitor

用于保留和查看协议层原始事件。

包含事件：

- `available_commands_update`
- `session/update` 中无法归类的事件
- `protocol.stderr`
- `permission_request` 的 raw payload
- unknown / raw events

展示规则：

- 默认隐藏。
- 仅显示入口，例如“协议调试 · 12 条隐藏事件”。
- 展开后按时间顺序显示 JSON。
- 不计入执行过程事件数量。

## 事件归类表

| 原始事件 | 标准类型 | 展示位置 | 默认展示 |
| --- | --- | --- | --- |
| `agent_message_chunk` | `assistant_message` | 聊天正文 | 是 |
| `agent_thought_chunk` | `thinking` | Timeline | 折叠 |
| `thinking` / `reasoning` | `thinking` | Timeline | 折叠 |
| `plan` | `plan_update` | Timeline | 展开 |
| `tool_call` | `tool_call` | Timeline | 展开 |
| `tool_call_update` | `tool_call` / `tool_result` / `file_diff` | Timeline / Diff | 展开 |
| `command_output_delta` | `command_output` | Timeline | 折叠或按大小折叠 |
| `available_commands_update` | `raw` | Traffic Monitor | 隐藏 |
| unknown event | `raw` | Traffic Monitor | 隐藏 |

## 数据流

后端仍然负责把 provider 原始事件标准化为 `AgentTimelineEvent` 并持久化到 `message.metadata.trace.events`。

前端新增一个展示层 reducer：

1. 接收 `AgentTimelineEvent[]`。
2. 按事件语义分成 `messageEvents`、`timelineEvents`、`debugEvents`。
3. 按 `toolCallId` 合并工具生命周期。
4. 从 tool update 中提取 diff，生成或合并为 `file_diff` 视图模型。
5. 输出给组件：
   - `messageContent`
   - `visibleTimelineEvents`
   - `hiddenDebugEvents`

## Tool Call 合并规则

合并主键：

- 优先使用 `payload.id`。
- 其次使用 `payload.tool_call_id`。
- 再其次使用事件 `id`。

合并字段：

- `title`：优先保留最具体的工具标题。
- `kind`：优先保留非空值。
- `input`：保留首次有效输入。
- `output`：保留最新有效输出。
- `status`：按生命周期推进，`failed` 优先级最高。
- `raw`：保留最近一次 raw，同时 Traffic Monitor 可查看全量 raw。

展示文案：

- started / in_progress：`调用工具 <title>`
- completed：`工具完成 <title>`
- failed：`工具失败 <title>`

## Diff 展示规则

diff 来源：

- `tool_call_update.content.type = "diff"`
- raw patch payload
- provider 已标准化的 `file_diff`

展示方式：

- 小 diff 在 timeline 中 inline 展示。
- 多文件或大 diff 显示摘要卡片，展开后查看详情。
- 每个 diff 显示文件路径、新增行、删除行。
- patch 行使用 `+` / `-` 样式区分。

## UI 结构

单条 agent 消息建议结构：

```text
规划师
这里是 assistant 正文回复，支持流式追加。

ACP 执行过程
5 条事件

[Thinking] 默认折叠
[Plan] 3 个步骤
[Tool] Read package.json · 完成
[Diff] 2 files changed
[Command] npm run build · 完成

[协议调试] 12 条隐藏事件
```

## 错误处理

- 如果只有 `assistant_message`，不显示 `AgentTimeline`。
- 如果只有 raw/debug event，显示“协议调试”入口，不显示空 timeline。
- 如果 tool call 缺少 `toolCallId`，按事件顺序展示，不强行合并。
- 如果 diff 无法解析 patch，退回 JSON/文本展示。
- 如果 provider 返回未知事件，保留 raw，不阻断正文流式渲染。

## 测试策略

前端测试：

- `assistant_message` 不渲染在 `AgentTimeline`。
- 只有 `assistant_message` 时 timeline 返回空。
- thinking 默认折叠。
- plan 展示中文状态。
- tool call + update 按 `toolCallId` 合并。
- diff 行正确标记新增和删除。
- raw / `available_commands_update` 默认进入调试隐藏区域。

后端测试：

- `agent_message_chunk` 仍标准化为 `assistant_message` 并能进入消息正文。
- `tool_call_update` 的 diff content 能标准化为 `file_diff`。
- unknown raw event 不丢失。

浏览器验证：

- 发送触发 ACP 工具调用的消息。
- 确认正文流式输出正常。
- 确认 timeline 不出现“助手回复”卡片。
- 确认工具调用不会因 update 产生多张重复卡片。
- 确认 raw/debug 默认隐藏。

## 验收标准

- 截图中的“助手回复 / 我 / 会 / 先 / 按 ...”不再出现在 `agent-timeline-summary`。
- timeline 数量不统计 `assistant_message` 和默认隐藏 raw/debug event。
- tool call 生命周期在 UI 中是一张稳定卡片。
- diff 有专门可读样式，不只是 JSON。
- 开发者仍能在 Traffic Monitor 查看完整 raw event。

## 实施建议

建议分三步实现：

1. 完成事件分层和 `assistant_message` 隐藏。
2. 增加 tool call lifecycle reducer 和 `available_commands_update` 调试分流。
3. 增加 diff panel / inline diff 强化展示。

第一步已经可以独立交付，后两步可继续迭代。
