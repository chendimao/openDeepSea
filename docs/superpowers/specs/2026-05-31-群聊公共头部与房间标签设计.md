# 群聊公共头部与房间标签设计

## 背景

群聊页面需要与首页等页面统一使用公共 header。当前群聊页仍保留独立的顶部导航职责，容易与公共导航重复。用户确认新的群聊导航方向为：公共 header 在所有页面通用，群聊页面在 header 下方增加房间 tab 栏，用于切换当前项目内的所有群聊。

本设计只覆盖公共 header、项目菜单、房间 tab 栏、房间排序与右键菜单相关能力，不处理气泡目录、气泡压缩包或其他无关原型。

## 已确认需求

- 群聊页面也使用公共 header。
- 首页公共侧边栏菜单统一放入 header 区域，header 为公共区域。
- 项目菜单改为鼠标经过展示所有项目，并保留点击展开能力以兼容触屏与键盘操作。
- 群聊页面主功能区域在 header 下方增加 tab 栏。
- tab 栏只展示当前项目内的所有群聊。
- 群聊按持久化的使用顺序从前往后排列。
- 置顶、最近使用顺序需要持久化到后端数据库。
- tab 右键菜单支持重命名、置顶/取消置顶、删除。
- 重命名使用轻量弹窗。
- tab 栏最右侧提供新增群聊按钮。
- 主功能区继续保持左侧任务工作区 60%、右侧聊天区域 40%。
- Apple 玻璃液化风格只属于 Apple 风主题；极简主题保持简单扁平。

## 选定方案

采用“公共 header + 独立房间 tab 栏”。

页面层级为：

```text
公共 Header
房间 Tab 栏
主功能区：左侧任务工作区 60% / 右侧聊天区 40%
```

公共 header 只承载全局导航、项目入口和设置等公共能力。房间 tab 栏是群聊页的局部导航，负责当前项目内房间的切换与管理。聊天区内部 header 只保留当前房间状态、模型、成员等聊天上下文信息，不承担页面级导航。

该方案比把房间 tab 合并进 header 更清晰，也比左侧房间列表更符合“header 下方 tab 栏”的既定方向，不会挤占任务工作区宽度。

## 前端结构

### AppShell

`AppShell` 的公共 header 在所有路由展示，包括群聊路由。公共 header 中的项目入口展示当前项目；鼠标经过时展开全部项目列表，点击项目后切换到该项目上下文。

项目菜单交互要求：

- hover 展示全部项目。
- 点击也可以展开或收起，兼容触屏。
- 当前项目有明确选中状态。
- 项目列表过长时允许滚动，不撑开 header。

### RoomPage

`RoomPage` 移除重复的页面顶部导航职责，在公共 header 下方插入 `RoomTabsBar`。主功能区保持现有左右布局：

```text
TaskWorkspacePanel：60%
ChatPanel：40%
```

### RoomTabsBar

新增 `RoomTabsBar` 组件，职责如下：

- 拉取当前项目内的房间列表。
- 按置顶与最近使用顺序渲染 tab。
- 切换 tab 并更新当前房间的最近打开时间。
- 提供新增房间按钮。
- 提供 tab 右键菜单。
- 提供轻量弹窗完成重命名。
- 房间过多时横向滚动，最右侧新增按钮保持固定可见。

tab 展示内容：

- 房间名称。
- active 状态。
- 置顶标识。
- hover 与 focus 状态。

## 数据模型

房间数据增加两个持久化字段：

```text
last_opened_at：最近打开时间，可为空
pinned_at：置顶时间，可为空，空值表示未置顶
```

排序规则：

```text
1. 已置顶房间在前
2. 置顶房间按 pinned_at 倒序，最近置顶的靠前
3. 未置顶房间按 last_opened_at 倒序
4. 没有 last_opened_at 的旧房间回退到 created_at 倒序
```

切换到房间时更新 `last_opened_at`。执行置顶时写入当前时间到 `pinned_at`。取消置顶时将 `pinned_at` 设为空。

## API 设计

后端需要补齐房间更新能力，并让列表接口返回新增字段。

### 列表

当前项目房间列表返回当前项目内所有房间，并包含：

```text
id
project_id
name
description
created_at
last_opened_at
pinned_at
```

后端可以直接按设计排序返回；前端也可以保留同样排序逻辑作为兜底。

### 创建

创建房间后，前端自动切换到新房间。新房间的 `last_opened_at` 应写入当前时间，使它进入最近使用序列靠前位置。

### 更新

房间更新接口支持以下字段的局部更新：

```text
name
last_opened_at
pinned_at
```

前端只通过该接口完成重命名、使用顺序更新、置顶与取消置顶。

### 删除

删除房间前前端必须二次确认。删除成功后：

- 如果还有其他房间，切换到排序后的第一个房间。
- 如果没有房间，展示空状态和新增按钮。
- 如果删除失败，当前 tab 保持不变并提示错误。

## 交互细节

### 切换房间

点击 tab 切换房间，并更新该房间的 `last_opened_at`。切换后主工作区和聊天区域都跟随当前房间刷新。

### 新增房间

tab 栏最右侧新增按钮创建新房间。创建成功后立即切换到新房间。新增按钮不随 tab 横向滚动，始终固定在右侧。

### 右键菜单

右键房间 tab 打开菜单：

```text
重命名
置顶 / 取消置顶
删除
```

菜单应支持键盘可达性。点击菜单外部、按 Escape、切换房间时关闭菜单。

### 重命名

重命名使用轻量弹窗。弹窗包含房间名称输入框、确认按钮、取消按钮。确认时校验非空名称，更新失败时弹窗保持打开并显示错误。

### 删除

删除使用确认弹窗，明确提示删除的房间名称。确认后执行删除。删除失败时显示错误，并保留当前房间。

## 主题与视觉

极简主题保持简单、扁平、低装饰：

- 轻边框。
- 低饱和背景。
- 清晰 active 状态。
- 不使用玻璃液化、强模糊、强高光。

Apple 风主题可以继续使用已有玻璃液化风格，但该风格不能泄漏到极简主题。

图标可以使用当前系统中更现代的 icon/svg，优先用于：

- 项目菜单展开。
- 置顶状态。
- 新增群聊。
- 右键菜单项。
- 删除危险操作。

## 错误处理

- 房间列表加载失败：tab 栏显示轻量错误状态，并保留主区域当前可用内容。
- 切换房间失败：保留当前房间并提示错误。
- 新增失败：在 tab 栏附近提示错误。
- 重命名失败：弹窗保持打开并显示错误。
- 置顶失败：tab 顺序回滚或重新拉取列表。
- 删除失败：当前 tab 保持不变并提示错误。

## 验证方式

实现完成后只执行一次最终构建：

```bash
npm run build
```

同时进行页面 smoke 验证：

- 群聊页面存在公共 header。
- 项目菜单 hover 可以看到全部项目。
- 当前项目内所有群聊显示在 tab 栏。
- tab 切换会更新最近使用顺序。
- 置顶后房间排在非置顶房间前。
- 取消置顶后回到最近使用排序。
- 右键菜单可重命名、置顶/取消置顶、删除。
- 重命名使用轻量弹窗。
- 删除需要二次确认。
- 新增按钮固定在 tab 栏最右侧，新增后自动切换到新房间。
- 极简主题没有 Apple 玻璃液化样式。
- 主功能区仍为左 60%、右 40%。

## 非目标

- 不引入气泡目录或气泡压缩包中的任何设计。
- 不重做群聊消息渲染体系。
- 不改变任务工作区与聊天区的核心业务逻辑。
- 不进行无关的大规模重构。
