# 会话页面后端接入补齐设计

## 背景

当前 `http://localhost:5173/projects/b6p57AAqxbcl` 的 Session OS 页面已经会请求 `GET /api/projects/:projectId/session-workspace`，并且后端能返回 active session、history records、messages、runs、evidence 和 status。问题不在于完全没有后端请求，而在于 Deepsea 会话页面仍保留大量 Stitch 还原稿中的静态数据和 mock fallback，导致多个模块看起来没有真实接入后端。

本设计补齐当前页面的后端接入能力，同时保持现有 Deepsea 视觉结构不重做。

## 目标

1. 让当前会话页面首屏可见信息全部来自后端数据或明确的空态。
2. 补齐页面已有按钮背后的后端能力：停止运行、重新执行、Compact discard、history 搜索筛选、目标契约编辑。
3. 去掉项目切换器、底部状态栏、工具调用、待提交变更、会话计划中的静态假数据。
4. 修复 session WebSocket 连接生命周期或刷新策略问题，确保运行时事件能驱动页面更新。
5. 保持当前页面布局、视觉语言和组件层级稳定，只做数据绑定、API 扩展和状态处理。

## 非目标

1. 不实现会话计划编辑、拖拽排序或状态切换；会话计划仅展示真实 `planItems`，为空显示空态。
2. 不实现通知中心、未读通知、通知详情。
3. 不实现底部导出按钮或导出当前 session 的新语义。
4. 不重做 Deepsea 视觉、不拆成新页面、不回退旧 rooms/tasks 工作流。
5. 不把旧群聊任务模型重新作为 Session OS 主状态源。

## 当前问题清单

### 已有后端但前端未完整消费

- `SessionWorkspacePage` 已获取 `SessionWorkspacePayload`，但 `SessionShellView` 中项目切换器仍使用静态 `projectSwitcherCards`。
- active session、history、messages、runs 能显示部分真实数据，但 provider/model、plan、tool rows、diff rows 仍混用 fallback。
- `/new`、`/compact`、`/status`、`/context`、resume、fork 已有 API 路径，但缺少完整 loading、错误态、成功后刷新和残留状态处理。
- session WebSocket 已有 subscribe 机制，但浏览器出现 `WebSocket is closed before the connection is established` warning，需要确认连接/卸载时序和订阅恢复。

### 需要补后端能力

- 项目切换器需要项目列表和每个项目最近 session/history 的聚合数据。
- 底部状态栏需要可展示的系统健康、索引状态、响应耗时、错误率、网络延迟和 token 消耗来源。
- 目标契约需要持久化 scope、risks、acceptance criteria 或等价结构，并提供更新 API。
- 停止运行需要 cancel endpoint 调用 `runRegistry`。
- 重新执行需要 retry endpoint，不能继续把“重试”按钮接到 `/status`。
- 工具调用需要标准化 rows，避免前端从 evidence payload 猜 path、duration 和状态。
- 待提交变更需要真实文件列表、增删行或 diff 摘要，不能只给 changed count。
- History 搜索筛选需要后端 query 参数或标准过滤结果。
- Compact 预览关闭需要 discard endpoint，避免后端一直残留 `previewed` 状态。

## 推荐方案

采用“后端聚合 ViewModel + 前端去 mock 绑定”的方式。

后端继续以 `GET /api/projects/:projectId/session-workspace` 作为主入口，扩展返回给页面直接消费的聚合字段。前端保持 `SessionShellView` 视觉结构，只把内部静态常量和 fallback 改成真实 payload 字段或空态。

这个方案的优点是：

- 页面只需要一次主请求即可渲染首屏，符合当前 Session OS 工作台模型。
- 不要求前端理解过多底层 evidence、git、runtime 细节。
- 可以用后端测试稳定数据 contract，降低页面继续出现静态假数据的风险。
- 对当前视觉改动最小，符合“不改页面设计”的边界。

## 后端设计

### 扩展 workspace payload

在现有 `SessionWorkspacePayload` 上新增可选聚合字段：

```ts
export interface SessionWorkspacePayload {
  project: Project;
  activeSession: SessionDetail;
  historyRecords: HistoryRecord[];
  status: StatusSnapshot;
  context: SessionContextManifest | null;
  evidence: SessionEvidenceEvent[];
  projectSwitcher: SessionProjectSwitcher;
  bottomStatus: SessionBottomStatus;
  contract: SessionContract;
  toolRows: SessionToolRow[];
  diffRows: SessionDiffRow[];
  historyFilters: SessionHistoryFilters;
}
```

### Project switcher

新增后端聚合类型：

```ts
export interface SessionProjectSwitcher {
  activeProjectId: string;
  projects: Array<{
    id: string;
    name: string;
    path: string;
    active: boolean;
    recentSessions: Array<{
      id: string;
      title: string;
      status: SessionStatus | HistoryRecord['status'];
      updated_at: number;
      href: string;
      source: 'session' | 'history';
    }>;
  }>;
}
```

数据来源：

- `projectRepo.list()` 获取项目。
- 每个项目取最近 active/archived sessions 和 history records，按 `updated_at` 或 `ended_at` 排序。
- `href` 使用 `/projects/:projectId` 或 `/projects/:projectId/sessions/:sessionId`。

### Bottom status

新增类型：

```ts
export interface SessionBottomStatus {
  health: 'ok' | 'warning' | 'error';
  healthLabel: string;
  indexStatus: 'ready' | 'building' | 'missing' | 'unknown';
  indexLabel: string;
  lastResponseMs: number | null;
  errorRate: number | null;
  networkLatencyMs: number | null;
  tokenUsage: {
    input: number;
    output: number;
    total: number;
  } | null;
}
```

最小实现：

- `health` 来自后端进程和最近 run 失败情况。
- `lastResponseMs` 从最近 completed run 的 `started_at/completed_at` 计算。
- `errorRate` 从最近 N 个 session runs 的 failed 比例计算。
- `networkLatencyMs` 暂时可为 `null`，前端显示 `--`。
- `tokenUsage` 从 evidence payload 中的 usage 事件汇总；没有则为 `null`。
- `indexStatus` 若当前没有索引服务，返回 `unknown`，不伪造“已建立”。

### Contract

新增类型和存储字段：

```ts
export interface SessionContract {
  sessionId: string;
  objective: string;
  scope: string | null;
  risks: string[];
  acceptanceCriteria: string[];
  updated_at: number;
}
```

推荐新增表 `session_contracts`，按 `session_id` 唯一：

- `session_id`
- `scope`
- `risks` JSON
- `acceptance_criteria` JSON
- `created_at`
- `updated_at`

`objective` 仍从 `session.current_goal ?? session.title` 派生，避免目标来源分裂。

新增 endpoint：

```text
PATCH /api/sessions/:sessionId/contract
```

请求体：

```ts
{
  scope?: string | null;
  risks?: string[];
  acceptanceCriteria?: string[];
}
```

写入后创建一条 `session_evidence_events`，类型为 `status` 或新增 `contract`。若不新增 evidence type，使用 `status` 并在 payload 中记录 `contract_updated: true`。

### Stop run

新增 endpoint：

```text
POST /api/session-runs/:runId/cancel
```

行为：

- 校验 run 存在。
- 只允许取消 `queued`、`running`、`retrying`。
- 调用 `runRegistry.cancel(runId)` 或现有等价能力。
- 更新 `session_runs.status = cancelled`。
- 广播 `session_run:updated` 和 `session_run:stream done=true`。
- 记录 evidence `blocker` 或 `status`，标题为 `Run cancelled`。

### Retry run

新增 endpoint：

```text
POST /api/session-runs/:runId/retry
```

行为：

- 校验原 run 存在并属于 session。
- 复用原 run 的 `prompt/provider/model/mode`。
- 新建一个 session run，通过 `runSessionAgent` 重新执行。
- 返回新 run 或 workspace payload。
- 广播 run created/updated/stream。

### Tool rows

由后端从 evidence 标准化，不让前端猜字段：

```ts
export interface SessionToolRow {
  id: string;
  action: 'read' | 'write' | 'edit' | 'exec' | 'browser' | 'tool';
  label: string;
  target: string;
  status: 'completed' | 'running' | 'failed' | 'unknown';
  durationMs: number | null;
  severity: SessionEvidenceSeverity;
  eventId: string;
  created_at: number;
}
```

映射规则：

- `file_read` -> `read`
- `file_diff` -> `edit`
- `test` / `build` -> `exec`
- `browser_check` -> `browser`
- `tool_call` / `tool_result` -> `tool`

### Diff rows

新增后端 git diff 摘要构建：

```ts
export interface SessionDiffRow {
  path: string;
  status: 'added' | 'modified' | 'deleted' | 'renamed' | 'untracked' | 'conflicted';
  additions: number | null;
  deletions: number | null;
  summary: string | null;
}
```

数据来源：

- `git status --porcelain`
- `git diff --numstat`
- 工作目录使用 `session.worktree_path ?? session.workspace_path ?? project.path`

没有变更时返回空数组，前端显示 clean 空态，不再返回假文件。

### History search/filter

扩展：

```text
GET /api/projects/:projectId/history-records?q=&status=&mode=
```

规则：

- `q` 匹配 title、summary、resume_brief。
- `status` 支持单值或逗号分隔。
- `mode` 支持 session mode。
- 默认仍返回全部，保持兼容。

### Compact discard

新增：

```text
POST /api/sessions/:sessionId/compact/discard
```

请求体：

```ts
{
  compaction_id: string;
}
```

行为：

- 校验 compaction 属于 session。
- 仅允许 `previewed` 状态 discard。
- 更新为 `discarded`。
- 记录 evidence `compact`，标题为 `Compact discarded`。

## 前端设计

### 数据绑定原则

1. 页面不再维护静态业务常量作为真实数据 fallback。
2. 后端字段缺失时显示空态或 `--`，不显示假项目、假工具、假性能数据。
3. 保持现有 className 和布局，不重做视觉。
4. 所有会写后端的按钮需要 loading、disabled 和错误 toast。

### 具体改动

- `TopCommandBar`
  - `activeProjectName` 改为 `payload.project.name`。
  - 项目切换器读取 `payload.projectSwitcher.projects`。
  - 最近会话按钮跳转真实 `href`。

- `BottomStatusBar`
  - 接收 `payload.bottomStatus`。
  - `networkLatencyMs/tokenUsage` 为 `null` 时显示 `--`。
  - 不再硬编码 `良好`、`已建立`、`1.2s`、`0.0%`、`45ms`、`1,242 tokens`。

- `ContractModule`
  - 读取 `payload.contract`。
  - `Edit` 打开轻量内联编辑或小弹层。
  - 保存调用 `PATCH /sessions/:sessionId/contract`。

- `PlanModule`
  - 只展示 `payload.activeSession.planItems`。
  - 为空显示“暂无会话计划”，不再 mock 三条计划。

- `RunModule`
  - 停止按钮调用 `POST /session-runs/:runId/cancel`。
  - 重试按钮调用 `POST /session-runs/:runId/retry`。
  - 无 active run 时按钮 disabled。

- `ToolsModule`
  - 使用 `payload.toolRows`。
  - 为空显示“暂无工具调用”。
  - 删除 `fallbackTools`。

- `DiffModule`
  - 使用 `payload.diffRows`。
  - 为空显示 working tree clean 空态。
  - 删除 `fallbackDiffs`。

- `HistoryRail`
  - 搜索框和筛选按钮调用扩展后的 history records API。
  - 搜索结果只更新 history 列表，不重载整个 workspace。

- `CompactPreviewSurface`
  - Discard 时调用 compact discard API。
  - 成功后关闭 overlay 并刷新 workspace。

- WebSocket
  - 保留 session subscribe。
  - 修复快速卸载/重新订阅导致的 warning。
  - 收到 session 事件后优先局部更新 query cache；若事件类型无法精确合并，再 invalidate workspace。

## API 兼容策略

- 新增字段都放在 `SessionWorkspacePayload` 顶层，旧字段不删除。
- 前端在迁移期允许字段缺失并显示空态，便于后端分步实现。
- 新 endpoint 均为增量新增，不改变现有 `/new`、`/compact/preview`、`/compact/apply`、`/status`、`/context`、resume、fork 语义。

## 风险与权衡

- workspace payload 会变大。当前页面已经依赖聚合 payload，继续扩展比拆成多个首屏请求更简单；后续可对 project switcher 和 history filter 单独分页优化。
- git diff 读取可能较慢。应限制命令 timeout，并在失败时返回空数组加 warning 状态，而不是阻塞整个 workspace。
- retry run 可能重复执行用户命令。后端需要明确记录 retry evidence，并在前端显示这是新 run。
- contract 数据模型新增后，旧 session 没有 contract。前端应从 session title/current_goal 派生 objective，并显示 scope/risks 空态。

## 自审

- 本设计不包含会话计划编辑、通知、导出。
- 本设计要求改前端源码，但限定为数据绑定、按钮接线、状态处理和去 mock，不改视觉布局。
- 所有需要写后端状态的功能都有对应 endpoint。
- 所有当前静态假数据都有替换来源或空态策略。
- 验证覆盖后端 contract、API、前端渲染和浏览器 smoke。
