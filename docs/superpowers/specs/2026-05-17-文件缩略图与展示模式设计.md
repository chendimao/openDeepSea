# 文件缩略图与展示模式设计

## 背景

文件列表页当前以单一列表展示项目文件，左侧只显示基础文件图标。聊天输入区的“选择项目文件”弹窗也使用单一列表，缺少图片缩略图和视觉浏览模式。用户需要在文件列表页和选择文件弹窗中都能看到缩略图，并能在列表模式与 card 模式之间切换。

## 目标

1. 文件列表页展示文件缩略图。
2. 聊天界面的选择文件 modal 展示文件缩略图。
3. 两处都支持列表模式和 card 模式切换。
4. 两处展示逻辑尽量复用，保持视觉和交互一致。

## 非目标

1. 不新增服务端缩略图生成能力。
2. 不为 PDF、文本、代码等非图片文件生成内容截图。
3. 不修改上传接口、数据库结构或文件存储结构。
4. 不改变现有文件搜索、筛选、上传、删除、下载和引用语义。

## 设计决策

采用轻量前端缩略图方案：

- `image/*` 文件使用现有 `ProjectFile.url` 渲染真实图片缩略图。
- 非图片文件使用扩展名或文件类型图标作为占位缩略图。
- 抽取共享文件展示组件，供 `FilesPage` 和 `FilePickerDialog` 复用。
- 展示组件支持 `list` 与 `card` 两种布局，由调用方控制当前模式。

该方案复用现有数据结构，改动范围集中在前端，能满足当前浏览与选择文件的主要体验目标。

## 用户体验

### 文件列表页

文件列表页默认保留列表模式。列表模式继续展示文件名、项目、大小、类型、创建时间、引用次数和最近引用群聊。左侧图标区域升级为缩略图区域：图片显示真实图片，非图片显示类型占位。

页面工具区新增展示模式切换按钮，用户可在列表模式和 card 模式之间切换。card 模式使用网格布局，更突出缩略图和文件名，同时保留预览、下载、删除等操作入口。

### 选择文件 Modal

选择文件 modal 默认使用列表模式，以维持紧凑选择效率。列表模式展示缩略图、文件名、大小、类型和创建时间，并保留“已添加”和待选择状态。

modal 中同样提供列表/card 切换。card 模式适合从图片或视觉文件中快速挑选。已选中文件以清晰选中态呈现，已添加文件保持禁用与“已添加”标识。

## 组件边界

建议新增共享组件，例如 `ProjectFileView` 或 `ProjectFileGrid`：

- 输入：`files`、`viewMode`、`selectedIds`、`disabledIds`、`variant`、`onPreview`、`onDownload`、`onDelete`、`onToggleSelect` 等按场景裁剪的 props。
- 职责：渲染缩略图、文件名、元信息、列表/card 布局、选中态和基础操作区。
- 不负责：加载文件数据、执行上传/删除 mutation、管理搜索筛选、管理 modal open state。

建议同时抽出 `ProjectFileThumbnail`：

- 输入：`file`、`size` 或场景 className。
- 图片文件渲染 `<img loading="lazy">`。
- 非图片文件显示扩展名标签和文件类型图标。

## 数据流

现有 API 返回的 `ProjectFile` 已包含 `mime_type`、`original_name`、`url`、`size`、`created_at` 等字段，足够支持轻量缩略图和元信息展示。

`FilesPage` 继续负责项目/群聊筛选、搜索、上传、删除 mutation 和预览 dialog。它将筛选后的文件列表传给共享展示组件。

`FilePickerDialog` 继续负责弹窗打开状态、项目文件加载、搜索、待添加列表和提交选择。它将可见文件、已添加文件和待选择状态传给共享展示组件。

## 错误与边界处理

1. 图片加载失败时回退到非图片占位缩略图。
2. 文件名过长时继续使用单行截断，card 模式中最多展示有限行数。
3. 空列表、加载中和搜索无结果沿用现有提示。
4. 移动端 card 模式使用单列或窄网格，避免文字和操作按钮重叠。
5. 选择 modal 中已添加文件保持不可再次选择。

## 验证方式

1. 运行 `npm run build`，确认 TypeScript 与 Vite 构建通过。
2. 人工检查文件列表页：
   - 图片文件显示真实缩略图。
   - 非图片文件显示类型占位。
   - list/card 切换正常。
   - 预览、下载、删除操作仍可用。
3. 人工检查聊天选择文件 modal：
   - 图片与非图片缩略图展示正确。
   - list/card 切换正常。
   - 选择、取消选择、已添加禁用和添加按钮计数正确。
4. 在移动端宽度检查两处布局不溢出、不重叠。

## 风险

1. 共享组件 props 过宽会增加维护成本。实现时应按当前两个调用场景最小化接口。
2. card 模式可能降低文件元信息扫描效率，因此默认模式应保留为列表。
3. 图片 URL 直接加载可能受文件大小影响。当前只做浏览器懒加载，不引入服务端缩略图缓存。
