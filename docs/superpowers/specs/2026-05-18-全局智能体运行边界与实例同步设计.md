# 全局智能体运行边界与实例同步设计

## 背景

当前智能体架构已经区分了两类对象：

- 全局智能体：`agents` 表中的可复用智能体定义，承担身份、职责与默认 ACP 配置。
- 群聊智能体实例：`room_agents` 表中的房间内运行副本，承担实际运行边界、会话、任务归属和房间级配置。

现有后端已经具备全局默认运行边界字段，包括 `default_runtime_backend`、`default_tool_policy`、`default_workspace_policy` 和 `default_memory_scope`。但 `/agents` 页面没有暴露这些字段，用户无法在全局智能体页面配置默认工具能力、可写目录和记忆范围。

同时，群聊实例已经有运行边界配置抽屉，但缺少“从全局默认值重新同步”的显式操作。全局默认值修改后不应自动覆盖已加入群聊的实例，否则会破坏房间内的本地定制。

## 目标

1. 在 `/agents` 页面暴露全局默认运行边界配置。
2. 全局默认值只影响未来加入群聊的实例，不自动同步到已存在实例。
3. 已加入群聊的实例可以手动同步当前全局默认运行边界。
4. 同步只覆盖运行边界，不覆盖身份、职责、规则、会话和任务历史。
5. 将 `memory_scope` 从展示字段收紧为实际记忆读取边界，提升“下次遇到同类问题复用经验”的可预期性。

## 非目标

- 不做全局批量同步。
- 不做字段级勾选同步。
- 不自动覆盖已加入群聊实例。
- 不同步 ACP session id、session label、任务归属、运行记录或历史消息。
- 不引入跨项目记忆继承。
- 不引入向量检索；语义检索可作为后续独立设计。

## 全局智能体默认运行边界

`/agents` 页面新增“默认运行边界”区域，复用当前 `AcpConfigPanel` 的配置语义，但面向全局默认值。

字段包括：

- 默认 ACP 后端：`default_acp_backend`
- 默认权限：`default_acp_permission_mode`
- 默认运行后端：`default_runtime_backend`
- 默认工具能力：`default_tool_policy.allowed`
- 默认可读/可写目录：`default_workspace_policy.read/write`
- 默认记忆范围：`default_memory_scope`

页面保存时，前端 `AgentInput` 需要带上这些字段，后端 `/agents` create/patch schema 需要接收并传给 `agentRepo.create/update`。这些字段已经在 repo 层存在，因此重点是补齐 API schema 和前端表单链路。

内置智能体仍允许恢复默认配置。恢复默认会回到源码模板中的运行边界；普通保存则写入用户当前配置。

## 群聊实例同步全局默认值

在 `AcpConfigPanel` 的运行边界区域增加“同步全局默认值”按钮。

按钮语义：

- 只对有关联 `global_agent_id` 的群聊实例可用。
- 如果实例没有关联全局智能体，按钮禁用或隐藏。
- 点击后弹确认说明：只同步运行边界，不修改名称、职责、规则、会话和任务记录。
- 同步成功后刷新 `room-agents` query，并提示“已同步全局默认运行边界”。

同步字段：

- `acp_enabled`
- `acp_backend`
- `acp_permission_mode`
- `runtime_backend`
- `tool_policy`
- `workspace_policy`
- `memory_scope`
- 后续若全局支持 `memory_max_context_chars`，也纳入同步。

不同步字段：

- `agent_name`
- `agent_role`
- `workflow_role`
- `preferred_user_name`
- `personality`
- `responsibilities`
- `rules`
- `acp_session_id`
- `acp_session_label`
- `capabilities`
- 任务、消息、运行历史

## API 设计

新增接口：

```http
POST /rooms/:roomId/agents/:agentId/sync-global-runtime-defaults
```

响应：

- `200`：返回更新后的 `RoomAgent`。
- `404`：room agent 不存在，或不属于当前 room。
- `409`：room agent 没有关联全局智能体。
- `400`：全局默认配置不合法，通常应由 schema 与 repo 预先避免。

后端 repo 新增方法：

```ts
syncRuntimeDefaultsFromGlobal(roomAgentId: string): RoomAgent | undefined
```

该方法读取当前 room agent 的 `global_agent_id`，再读取对应全局 agent 的默认运行边界字段，写回当前 room agent。对内置智能体也使用“当前全局默认值”，而不是重新套源码内置模板，以尊重用户在 `/agents` 中修改过的全局默认值。

## 记忆范围收紧

当前 `memory_scope` 已进入 runtime profile，但记忆注入仍主要基于项目、房间、智能体和任务上下文统一加载。后续应让 `memory_scope` 真正控制读取范围。

建议语义：

- `project`：读取项目记忆、当前房间记忆、当前智能体记忆和当前任务记忆；允许跨房间相关记忆检索。
- `room`：读取当前房间记忆、当前智能体记忆和当前任务记忆；不读取跨房间相关记忆。
- `agent`：读取当前智能体记忆和当前任务记忆；不读取房间共享记忆和跨房间记忆。
- `task`：只读取当前任务记忆。
- `none`：不注入长期记忆。

普通对话自动沉淀仍默认写入 `project` 或 `room` 记忆。后续若要让“某个智能体自己学会某类问题”，可单独增强为按 `memory_scope` 写入 `agent` 记忆，但这不放入本次第一版。

## 数据流

### 新实例加入群聊

1. 用户从全局智能体库拉入智能体。
2. `roomAgentRepo.addFromGlobalAgent` 从 `agents.default_*` 复制运行边界。
3. 新的 room agent 获得当前全局默认运行边界。
4. 之后全局默认值变更不影响该 room agent。

### 已有实例手动同步

1. 用户在群聊智能体配置抽屉点击“同步全局默认值”。
2. 前端调用同步接口。
3. 后端校验 room agent 与 room 归属。
4. 后端读取关联全局 agent 的当前默认运行边界。
5. 后端覆盖 room agent 的运行边界字段。
6. 前端刷新 room agent 列表与当前抽屉状态。

### 智能体运行时读取记忆

1. `respondAsAgent` 解析 room agent 的 runtime profile。
2. 根据 `memory_scope` 选择记忆读取范围。
3. 将允许范围内的记忆注入 prompt。
4. 智能体回答完成后，若自动沉淀开启且运行成功，继续异步提取长期记忆。

## 错误处理

- 无全局智能体关联：同步接口返回 `409`，前端提示“该智能体没有全局默认值可同步”。
- 全局智能体已删除：理论上外键限制应避免；如果出现，返回 `409` 或 `404`。
- 同步期间实例被移出群聊：返回 `404`。
- 路径不合法：复用现有 workspace boundary path 校验，禁止绝对路径和 `..`。
- 记忆读取失败：沿用现有安全降级，记录 warning 并继续执行，不阻断智能体回复。

## 测试策略

后端测试：

- `/agents` create/patch 能保存默认运行边界字段。
- 新拉入群聊的 room agent 会复制当前全局默认运行边界。
- 修改全局默认值不会自动影响已存在 room agent。
- 同步接口只覆盖运行边界字段，不覆盖身份、职责、workflow role 和 session。
- 无 `global_agent_id` 的 room agent 调用同步接口返回 `409`。
- `memory_scope` 各取值对应正确的记忆读取范围。

前端验证：

- `/agents` 页面可编辑并保存默认运行边界。
- 群聊实例抽屉可点击“同步全局默认值”。
- 同步成功后抽屉显示最新运行边界。
- 无全局关联的实例不展示或禁用同步按钮。

最低交付验证：

- `npm run build`
- 相关后端单测
- 前端手动 smoke：全局修改默认值、新拉入生效、旧实例不变、旧实例手动同步后生效。

## 后续增强

- 在群聊实例抽屉显示“与全局默认值不同”的差异提示。
- 增加字段级同步。
- 增加全局页面“被 N 个群聊使用”的只读引用列表。
- 增加项目级经验库和语义检索，提升相似问题复用率。
- 自动把明确的解决方案沉淀为 agent scope 经验。
