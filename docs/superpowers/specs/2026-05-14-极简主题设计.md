# 极简主题设计

## 背景

当前 OpenClaw Room 前端已经支持 `light`、`dark`、`console` 三种主题。现有默认视觉偏玻璃拟态：包含渐变背景、半透明 surface、模糊、发光、大阴影和较圆的控件。用户希望新增一个参考 `/Users/chendimao/Library/Mobile Documents/com~apple~CloudDocs/Downloads/ai-collab-dev-system-2.html` 的极简主题。

参考稿的核心视觉语言是浅色本地控制台：近白背景、纯白面板、细分隔线、紧凑信息密度、monospace 数据标签、低装饰和明确的 active 状态。

## 目标

新增一个可随意切换的独立主题，显示文案为“极简”，技术值建议为 `minimal`。

该主题只改变样式风格，不改变页面布局、DOM 结构、路由、数据获取、WebSocket、业务状态或交互行为。

## 非目标

- 不重写 `AppShell`、`RoomPage`、`ProjectPage` 或 `DashboardPage` 的页面结构。
- 不引入参考稿中的四栏 rail/sidebar/main/board 新布局。
- 不改变现有 `light`、`dark`、`console` 主题的视觉规则。
- 不新增业务功能、API、数据库字段或 WebSocket 事件。
- 不调整任务、消息、Agent、工作流等业务模型。

## 主题入口

在 `ThemeMode` 中增加 `minimal`。

主题切换控件增加第四个选项：

- 值：`minimal`
- 文案：`极简`
- 图标：复用现有主题图标体系，优先选择表达界面/面板/控制台的 lucide 图标

切换后继续使用现有机制写入：

- `document.documentElement.dataset.theme = theme`
- `localStorage.setItem('openclaw-room-theme', theme)`

切回其他主题时，应恢复对应主题原样。

## 视觉规范

### 基础 token

`[data-theme="minimal"]` 下定义独立 token：

- 背景：近白或极浅冷灰
- surface：纯白
- raised surface：纯白或极浅灰
- 文本：深灰黑
- muted 文本：中性灰
- border：浅灰细线
- primary/accent：冷蓝，仅用于 active、链接、关键状态和主按钮
- success/warning/danger：保留现有语义色，但降低饱和度或减少发光

### 装饰收敛

极简主题应移除或显著降低：

- 大面积渐变
- 背景光斑
- 玻璃模糊
- 发光阴影
- 浮动卡片阴影
- 过大的圆角

保留清晰的层级关系，但主要通过边框、背景明度和留白表达。

### 组件风格

所有覆盖必须挂在 `[data-theme="minimal"]` 下，避免影响原主题。

建议覆盖的现有类包括：

- `.liquid-backdrop`
- `.glass-sidebar`
- `.liquid-logo-small`
- `.glass-search`
- `.theme-toggle`
- `.theme-toggle-option`
- `.gateway-status-button`
- `.sidebar-link`
- `.recent-project-link`
- `.glass-project-card`
- `.metric-tile`
- `.workspace-root`
- `.workspace-toolbar`
- `.toolbar-back`
- `.toolbar-tab`
- `.glass-button`
- `.workbench-panel`
- `.inspector-panel`
- `.chat-stream-header`
- `.task-board-toolbar`
- `.inspector-header`
- `.segmented-control`
- `.message-bubble`
- `.user-message`
- `.agent-run-console`
- `.composer-box`
- `.task-column`

覆盖方向：

- 面板：纯白背景、1px 浅灰边框、无 blur、无大阴影
- 控件：低圆角或轻微圆角，边框优先，hover 只改变背景或文字色
- active 状态：使用左侧细色条、浅灰底或冷蓝文字，不使用大面积发光
- 消息：减少 bubble 卡片感，偏向列表式阅读；用户消息不使用强渐变
- 任务栏和 inspector：用细线分隔标题、列表和内容区域
- 数据标签、路径、计数：保持 monospace 和 tabular numeric

## 架构与边界

实现上采用“主题 token + scoped CSS 覆盖”的方式。

核心边界：

- TypeScript 只新增主题枚举值和主题切换选项。
- CSS 只新增 `[data-theme="minimal"]` 相关规则。
- 不修改 JSX 结构来适配主题。
- 不新增运行时主题分支逻辑，除非只是为了让主题选项可选。

该方式能确保极简主题是可逆的：用户切回 `light`、`dark` 或 `console` 后，不应保留极简主题的视觉副作用。

## 风险

主要风险是现有样式中大量组件类共享玻璃拟态假设，单纯覆盖 token 可能不足以消除所有发光和模糊。因此实现时需要按组件类补充 scoped 覆盖。

另一个风险是主题切换控件空间变窄。新增第四个选项后需确认文本不会挤压或溢出。

## 验证方式

最低验证：

- 运行 `npm run build`，确认 TypeScript 与 Vite 构建成功。

实现后建议验证：

- 本地启动前端。
- 在 `浅色`、`深色`、`控制台`、`极简` 四个主题间来回切换。
- 检查原主题切回后没有残留极简样式。
- 检查主要页面：Dashboard、Project、Room。
- 检查 Room 页面中的聊天、任务栏、inspector、composer、主题切换控件没有文本溢出或明显重叠。

## 批准结论

用户已批准方案 C 的收敛版本：页面布局和结构不改变，只新增可随意切换的独立极简主题，并且不影响原有样式。主题显示文案确定为“极简”。
