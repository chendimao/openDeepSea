# 任务详情正文与 ACP 流转时间线合并设计

## 背景

当前任务详情面板中，任务正文由独立的“描述”区块展示，ACP 流转记录则在 `timeline` / `diff` / `logs` 等视图中展示。用户在查看任务详情时，需要在正文和 ACP 流转记录之间来回对应上下文，阅读顺序被拆开。

本次改动目标是让任务正文与 ACP 流转记录在任务详情的时间线中按发生时间从上到下展示，形成单一阅读流。

## 目标

1. 在任务详情的 `timeline` 视图中展示任务正文条目。
2. 任务正文条目与现有任务事件按时间升序合并展示。
3. 移除任务详情中独立的“描述”区块，避免正文重复出现。
4. 保留任务基础信息、执行会话、状态编辑、指派智能体等现有操作能力。
5. 保持后端接口和数据库结构不变。

## 非目标

1. 不改 ACP 事件采集、存储或归一化逻辑。
2. 不改 `plan`、`diff`、`logs` 视图的信息模型。
3. 不新增任务正文编辑能力。
4. 不从 source message 反查更精确的正文时间；本次使用 `task.created_at` 作为正文条目时间。

## 设计方案

采用“Timeline 视图内合并”的轻量方案。

任务详情面板上方继续展示基础信息和执行会话。原独立“描述”区块移除。进入 `timeline` 视图时，前端将 `task.description` 构造成一个只读正文条目，并与现有 `TaskEvent` 列表合并排序后渲染。

正文条目属于 `chat` layer。现有 layer 过滤器已包含 `chat` 的状态字段，但当前任务详情过滤按钮只展示 `activity`、`timeline`、`runtime`、`diff`。本次不新增过滤按钮，只遵循已有 `layerVisibility.chat`：当 `chat` 可见时展示正文条目，不可见时隐藏正文条目。

## 组件边界

### `TaskDetailPanel.tsx`

- 新增一个内部时间线条目模型，统一表示正文条目和事件条目。
- 新增构建函数，将 `task.description` 转为正文时间线条目。
- 修改 `timeline` 视图的数据来源：由单纯 `timelineEvents` 改为“正文条目 + timelineEvents”的合并结果。
- 移除独立的 `taskDetail.description` section。
- 保持 `TaskEventTimeline` 对事件条目的渲染职责，不把正文伪装成后端 `TaskEvent`。

### 测试

在现有 `TaskDetailPanel.render.test.tsx` 或邻近测试中补充覆盖：

1. 有任务正文和事件时，正文与事件按时间顺序渲染在同一时间线容器中。
2. `layerVisibility.chat` 为 `false` 时，正文条目不渲染。
3. 独立“描述”区块不再出现在任务详情主体中。

## 数据流

1. `TaskDetailPanel` 接收当前 `task` 与 task events。
2. `selectTaskDetailEvents` 仍负责按 layer 和视图筛选事件。
3. `timeline` 视图渲染前，将正文条目和 `timelineEvents` 合并。
4. 合并列表按 `created_at` 升序排序；时间相同时正文条目排在同时间事件之前，确保用户先看到任务上下文。
5. 渲染层根据条目类型展示：
   - 正文条目：标题为“正文”，内容为 `task.description`；空正文不生成条目。
   - 事件条目：沿用现有事件标题、摘要、payload 详情展示。

## 空态与边界

- `task.description` 为空时，不生成正文条目。
- 若 timeline 事件也为空，则继续展示现有空态。
- 若 `chat` layer 被隐藏，正文条目隐藏；如果其他可见事件为空，则展示当前 timeline 空态。
- 长正文保持 `whitespace-pre-wrap` 和合理行高，避免压缩可读性。
- 不改变 diff / runtime payload 的截断策略。

## 风险

1. 任务详情原“描述”位置消失后，用户需要进入 `timeline` 视图才能看到正文。该风险可接受，因为本次明确目标是把正文纳入时间线阅读流。
2. 使用 `task.created_at` 作为正文时间可能不如 source message 时间精确。由于后端当前任务详情接口已直接提供任务创建时间，本次不扩大接口范围。
3. `chat` layer 当前没有在任务详情过滤按钮中显式展示，正文隐藏状态只能受上层默认值影响。为避免扩大 UI，本次不新增按钮。

## 验证方式

1. 运行前端相关渲染测试，确认正文与事件合并展示。
2. 运行 `npm run build`，确认 TypeScript 和前端构建通过。
3. 如本地服务可用，打开任务详情页手动检查：
   - 独立描述区块已消失。
   - `timeline` 视图中正文和 ACP 流转记录按时间从上到下展示。
   - `plan`、`diff`、`logs` 视图仍能正常切换。
