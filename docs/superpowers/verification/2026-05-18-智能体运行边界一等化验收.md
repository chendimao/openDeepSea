## 最终审查结论

- `dispatcher.ts` 通过 `resolveAgentRuntimeProfile()` 注入 runtime context，并将 `runtimeProfile.writableDirs` 传给 ACP adapter。
- `agent-runtime.ts` 会解析相对路径到项目内绝对路径，拒绝空路径、绝对路径逃逸、`..` 段和项目根外路径。
- Reviewer / Acceptor 内置模板默认 `read-only`，`workspace_policy.write` 为空。
- Backend Executor / Frontend Executor 默认分别限制到 `packages/backend` 与 `packages/frontend`。
- 工作流执行选择会要求写任务具备 `write_files`、非 `read-only` permission，且所有 `scopeWrite` 被 writable workspace 覆盖。
- API 更新运行边界时校验 workspace policy path，不回传 OpenAI API Key 等 secret。
- 旧数据通过 repo normalization 与内置模板迁移逻辑保留兼容默认值。

## 已知限制

- 前端构建仍有大 chunk warning，和本次运行边界改动无直接关系。
- OpenCode adapter 当前不消费 `acpWritableDirs`，只能依赖其自身权限模型；Codex 与 Claude Code 已使用解析后的可写目录。
