# 去除 OpenClaw 依赖真实浏览器验收

日期：2026-05-16

## 环境

- 前端：`http://localhost:5175/`
- 后端：`http://localhost:7331/`
- 后端启动参数：`LANGGRAPH_WORKFLOW_ENABLED=1`
- 浏览器工具：Playwright CLI / Chromium
- ACP CLI：Codex `0.130.0`，OpenCode `1.15.0`
- 模型：OpenAI-compatible endpoint，模型 `gpt-5.5`
- API key：已在 UI 模型配置中保存；本文档不记录密钥明文

## 覆盖范围

- 模型配置页面可打开，Base URL、模型与 API key 保存状态可见。
- 房间页稳定测试定位点可见：`room-page`、`chat-scroll-container`、`chat-composer`、`task-panel`、`memory-panel`。
- 页面发送 ACP 消息，Codex ACP agent 运行完成并返回聊天回复。
- `/task` 在聊天中创建任务，任务出现在任务列表，刷新后不重复。
- 手动新建 `ask_user` 任务只创建任务并写入聊天记录，不自动启动工作流。
- 用户从任务列表点击启动工作流后，创建 workflow 并写入聊天记录。
- 自动记忆：手动任务创建写入 task scope 的 `task_summary` 记忆。
- `auto_recommended` 任务创建后自动进入 LangGraph 工作流流转。
- 移动视口下聊天 composer 在可视区域内，聊天滚动容器拥有滚动条且 `overflow=auto`。
