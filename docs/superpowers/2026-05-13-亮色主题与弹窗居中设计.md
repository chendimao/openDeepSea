# 亮色主题与弹窗居中设计

## 目标

为 OpenClaw Room 增加 light/dark 双主题能力，默认使用 Harbor Light 亮色主题，同时保留当前暗色主题作为可回退选项。修复所有基于共享 `DialogContent` 的弹窗偏右下问题，确保新建任务、添加项目、新建聊天室、邀请 Agent 等弹窗都在视口正中。

## 设计选择

采用双主题切换，而不是只替换默认变量。亮色主题作为默认值写在 `:root`，暗色主题保留在 `.dark` 或等价主题选择器下。这样首次打开就是亮色，同时用户可以切回当前暗色视觉，刷新后保持上次选择。

视觉方向采用 Harbor Light：浅青白背景、白色面板、青灰边框、珊瑚主色和青蓝 accent。整体保持工作台产品的克制密度，不做营销式大面积装饰。

## 组件与状态

- `packages/frontend/src/index.css`：承载亮色默认 token、暗色 token、surface、overlay、scrollbar、code block 等基础样式。
- `packages/frontend/src/components/AppShell.tsx`：管理主题状态，初始化时读取 `localStorage`，无记录时使用 `light`；切换时更新根节点 class 和持久化值。
- `packages/frontend/src/main.tsx`：让 `Toaster` 跟随当前主题，避免亮色界面里出现暗色 toast。
- `packages/frontend/src/components/ui/Dialog.tsx`：调整弹窗动画和定位，避免动画 keyframes 覆盖居中 transform。

主题切换入口放在左侧项目栏底部的设置区域附近，使用明确的 sun/moon 图标按钮或图标加短文案。该入口不引入路由、全局 store 或后端配置。

## 弹窗居中修复

当前弹窗偏移的根因是 `DialogContent` 同时使用 Tailwind 的 `-translate-x-1/2 -translate-y-1/2` 和通用 `.fade-up` 动画；`.fade-up` keyframes 写入 `transform: translateY(...)`，会覆盖居中 transform。

修复方式：为 modal 使用独立动画，动画只改变 `opacity` 与 `scale`，或在 keyframes 中完整保留 `translate(-50%, -50%)`。优先选择独立 modal 动画，避免影响抽屉、面板等仍使用 `.fade-up` 的组件。

## 验收标准

1. 默认打开应用时为 Harbor Light 亮色主题。
2. 主题切换后可回到暗色主题，并在刷新后保持选择。
3. 新建任务、添加项目、新建聊天室、邀请 Agent 等共享 Dialog 弹窗在桌面与移动视口中都居中显示。
4. Toast、输入框、按钮、卡片、命令菜单和任务面板在亮色与暗色下均保持可读。
5. 执行 `npm run build` 通过。

## 非目标

- 不新增后端主题配置。
- 不做完整设计系统重构。
- 不新增多套主题调色板。
- 不重写页面布局与业务流程。

## 风险与处理

- 亮色主题下局部硬编码深色背景可能导致突兀：通过搜索 `bg-[var(--color-bg)]`、固定 hex 和 surface 使用点做定向修正。
- Toast 主题如果无法直接从 `AppShell` 状态传入，需要在根组件提升主题状态或用轻量自定义事件同步。
- 弹窗动画修复必须避免破坏侧边抽屉的进入动画，因此只改共享 Dialog 的动画类，不全局改 `.fade-up` 行为。
