# 文件缩略图与展示模式 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为文件列表页和聊天选择文件 modal 增加图片缩略图、非图片类型占位，以及列表/card 两种展示模式。

**Architecture:** 新增一个前端共享展示组件负责文件缩略图、列表/card 布局、选择态和操作区。`FilesPage` 继续管理筛选、上传、删除和预览；`FilePickerDialog` 继续管理弹窗、搜索和待选文件，二者只把展示所需数据和回调传给共享组件。

**Tech Stack:** React 18、TypeScript、Vite、Tailwind 工具类、现有 `index.css` 样式体系、lucide-react 图标、@tanstack/react-query。

---

## Scope Check

本计划只覆盖前端展示层。不会修改后端 API、数据库、上传逻辑或文件存储结构。当前工作区存在多处无关未提交改动，实现时必须只修改本计划列出的文件，提交时使用 pathspec 限定本功能文件。

```tsx
<ProjectFileView
  files={visibleFiles}
  mode={viewMode}
  variant="picker"
  isSelected={(file) => pendingIds.includes(file.id)}
  isDisabled={(file) => selectedSet.has(file.id)}
  onToggle={(file) => {
    if (selectedSet.has(file.id)) return;
    setPendingIds((current) =>
      current.includes(file.id)
        ? current.filter((id) => id !== file.id)
        : [...current, file.id],
    );
  }}
  getMeta={(file) => (
    <>
      <span>{formatFileSize(file.size)}</span>
      <span>{file.mime_type}</span>
      <span>{formatRelativeTime(file.created_at)}</span>
    </>
  )}
  getState={(file) => {
    const alreadySelected = selectedSet.has(file.id);
    const pending = pendingIds.includes(file.id);
    if (alreadySelected) return <span className="file-picker-state">{t('files.picker.added')}</span>;
    if (pending) return <Check className="h-4 w-4 text-[var(--color-primary)]" />;
    return null;
  }}
/>
```
