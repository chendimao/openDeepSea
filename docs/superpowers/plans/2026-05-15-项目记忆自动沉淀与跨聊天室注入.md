# 项目记忆自动沉淀与跨聊天室注入 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 完善 A 阶段项目级记忆闭环：聊天室与手动任务能可靠沉淀多条记忆，项目内其他聊天室能自动检索并注入相关经验。

**Architecture:** 保持现有 `memory_entries` 作为唯一记忆存储；自动沉淀继续使用 OpenClaw Gateway LLM 提取，但每条候选使用稳定子 source id，避免同一触发源只保存一条。跨聊天室引用先用 SQLite 中的确定性关键词相关性检索，不引入 embedding；Agent prompt 注入时合并当前上下文记忆与项目内其他聊天室相关记忆，并保留来源 scope。B 阶段只补计划，不实现审计 UI。

**Tech Stack:** Node.js、TypeScript、better-sqlite3、Express、node:test、React 现有记忆面板。
