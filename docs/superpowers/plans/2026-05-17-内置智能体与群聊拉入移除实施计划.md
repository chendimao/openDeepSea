# 内置智能体与群聊拉入移除 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将内置智能体纳入全局智能体库并支持编辑，群聊从全局库多选拉入智能体，移除群聊智能体时保留历史对话、运行记录和任务可追溯性。

**Architecture:** 后端以 `agents` 作为唯一智能体定义源，新增内置标记和默认配置恢复能力；`room_agents` 改为软移除，默认查询只返回 active 智能体。前端 `/agents` 展示内置与用户智能体，群聊邀请弹窗改为搜索式多选提交，移除智能体时根据运行中记录和未完成任务给出约束或转交流程。

**Tech Stack:** Node.js、Express、SQLite、TypeScript、React 18、TanStack Query、Vite、Tailwind 风格工具类。

### Task 3: 后端 API

**Files:**
- Modify: `packages/backend/src/routes.ts`
- Modify: `packages/frontend/src/lib/types.ts`
- Modify: `packages/frontend/src/lib/api.ts`

- [x] **Step 1: 调整全局智能体 API**

`DELETE /api/agents/:agentId`：
- 内置智能体返回 409 `{ error: 'builtin agent cannot be deleted' }`
- 被引用继续返回 409 references

新增可选：
- `POST /api/agents/:agentId/restore-defaults`，仅内置智能体可用。

- [x] **Step 2: 调整群聊拉入 API**

新增或扩展：
- `POST /api/rooms/:roomId/agents/batch`
- body: `{ global_agent_ids: string[] }`
- 已加入 active 智能体跳过或返回现有记录，不重复创建
- 已软移除的同 `agent_id` 重新拉入时恢复 `left_at=NULL`，并刷新当前全局定义快照字段

- [x] **Step 3: 调整群聊移除 API**

`DELETE /api/rooms/:roomId/agents/:agentId` 支持 query/body：
- `task_action?: 'unassign' | 'transfer'`
- `transfer_to_room_agent_id?: string`

行为：
- 有运行中 run：409
- 有未完成任务且无策略：409，返回任务数量
- 策略合法后软移除，并广播 `room:agent_left`
