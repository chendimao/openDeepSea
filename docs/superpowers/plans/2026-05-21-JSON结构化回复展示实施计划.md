# JSON 结构化回复展示 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将消息中的 `json` 代码块默认渲染为结构化中文视图，并支持任务准备语义卡片、原文切换与一键复制。

**Architecture:** 在 `MessageContent.tsx` 内新增 JSON 解析、字段中文映射、通用递归 Viewer 与 `task_readiness` 语义摘要卡片；非 JSON、解析失败和原文模式继续复用现有 `CodeBlock`。样式集中追加到 `index.css` 的消息展示区域，i18n 增加中英文标签。

**Tech Stack:** React 18、TypeScript、lucide-react、现有 i18n、CSS utility/theme variables、`npm run build`。

### Task 2: 实现 JSON Viewer 与语义卡片

- [x] **Step 1: 新增 JSON 解析分支**
  - 只在 `language` 为 `json` / `application/json` 且 `JSON.parse` 成功时启用。
  - 解析失败回退 `CodeBlock`。

- [x] **Step 2: 新增通用递归结构化视图**
  - 对 object/array/primitive 递归展示。
  - 字段名通过映射显示中文，同时保留原字段名作为小字辅助。
  - 字符串换行与长文本使用 `overflow-wrap: anywhere`，避免撑宽消息。

- [x] **Step 3: 新增 `task_readiness` 语义摘要**
  - 显示标题、就绪状态、置信度、推荐模式、执行意图、缺失问题数量。
  - 语义卡片只作为摘要，不替代下方完整 JSON Viewer。

- [x] **Step 4: 增加结构化/原文切换与复制反馈**
  - 默认 `structured`。
  - 原文模式保留在 JSON 容器内，可切回结构化。
  - 复制按钮复制原始 JSON。
