# 执行型智能体项目根目录可写 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让所有内置 `workflow_role: 'executor'` 智能体默认具备当前项目根目录写权限，同时保留非执行角色只读边界。

**Architecture:** 修改内置模板作为单一事实源，并通过 runtime profile version 从 `3` 升到 `4` 触发现有内置全局智能体和房间智能体迁移。测试从模板、全局 agent 种子、房间模板创建三个入口验证新默认边界。

**Tech Stack:** TypeScript, Node.js test runner, better-sqlite3, Express route tests.

---

const isDocsOnlyDefault = existing.acp_permission_mode === 'workspace-write'
