# 任务流转三分区展示与智能体 Tabs 重构实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将工作流任务展示拆分为计划区、流转时间线和智能体结果区，避免把代码审查压在同一张计划表内，并让 Tabs 数量与内容语义清晰。

**Architecture:** 以 `WorkflowTaskBubble` 作为编排入口，新增或重构一个专门的流转区组件承载执行、审查、验收阶段的顺序展示；`WorkflowTaskTable` 只保留计划子任务；`WorkflowAgentTabs` 负责按智能体聚合执行记录与产物，并明确显示数量语义。样式集中在 `packages/frontend/src/index.css`，测试覆盖 bubble、tabs 和流转展示的核心文案与状态映射。

**Tech Stack:** React 18, TypeScript, Radix Tabs, Tailwind utilities, React Testing Library/static render, existing frontend CSS.
