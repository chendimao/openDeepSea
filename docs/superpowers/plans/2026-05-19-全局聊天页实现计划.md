# 全局聊天页 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 构建一个侧边栏入口的全局聊天页，支持独立持久化会话、默认检索全局/项目记忆和非敏感配置摘要，并允许手动保存消息为全局记忆。

**Architecture:** 后端新增 `global_chat_sessions` 和 `global_chat_messages`，并扩展 memory scope 支持 `global`。全局聊天服务负责保存消息、构建检索上下文、调用现有系统模型配置并记录引用来源；前端新增 `/chat` 页面和侧边栏入口，复用现有 API、消息渲染和 Tailwind 风格。

**Tech Stack:** Node.js、Express、better-sqlite3、LangChain ChatOpenAI、React 18、TypeScript、Vite、TanStack Query、Tailwind。

- [ ] **Step 1: 写全局记忆失败测试**

新增测试断言：

```ts
test('global memories do not require a project and are returned for global chat context', () => {
  const memory = memoryRepo.create({
    scope: 'global',
    memory_type: 'preference',
    title: '偏好',
    content: '用户喜欢默认自动检索。',
    source_type: 'manual',
    pinned: true,
  });

  const context = memoryRepo.listForGlobalChatContext({ prompt: '检索偏好', limit: 10 });
  assert.equal(memory.project_id, null);
  assert.ok(context.some((entry) => entry.id === memory.id));
});
```

- [ ] **Step 2: 运行测试确认失败**

Run: `node --import tsx --test packages/backend/src/global-chat.test.ts --test-name-pattern "global memories"`

Expected: FAIL，原因是 `MemoryScope` 不支持 `global` 或 DB trigger 拒绝空 project。

- [ ] **Step 3: 实现全局记忆支持**

修改：

- `MemoryScope = 'global' | 'project' | 'room' | 'agent' | 'task'`
- DB trigger 允许 `scope = 'global'` 时 `project_id/room_id/room_agent_id/task_id` 均为 NULL。
- `memoryRepo.create` 对 global 跳过 project ownership 校验。
- `memoryRepo.listForGlobalChatContext({ prompt, limit, maxChars })` 返回 pinned global + 关键词相关 global/project/room/task 记忆。
- `memoryRepo.search` 如后续要暴露全局查询，则支持可选 `projectId`；第一版可仅供内部检索使用。

- [ ] **Step 4: 运行测试确认通过**
