# 群聊流式逐字渲染 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让正在流式输出的群聊 agent 消息以“普通文字逐字、代码/日志块级追加”的方式展示，并在输出期间显示闪烁光标。

**Architecture:** 保持后端 `message:stream` 协议与 React Query 中的真实 `Message.content` 不变，在前端增加临时展示缓冲层。纯文本分类与逐字队列逻辑拆到可测试的 `streamingDisplay` 工具，`RoomPage` 负责把 WebSocket chunk 投递给 hook，`MessageContent` 只负责展示内容与光标。

**Tech Stack:** React 18、TypeScript、Vite、TanStack Query、Node 内置 `node:test`、CSS。

import assert from 'node:assert/strict';
import test from 'node:test';
import {
  classifyStreamingChunk,
  createStreamingDisplayState,
  enqueueStreamingChunk,
  flushStreamingDisplay,
  tickStreamingDisplay,
} from './streamingDisplay.js';

test('普通中文文本进入逐字队列', () => {
  const state = enqueueStreamingChunk(createStreamingDisplayState(), '你好世界');
  assert.equal(state.displayed, '');
  assert.deepEqual(state.queue, ['你', '好', '世', '界']);
});

test('代码块内 chunk 块级追加', () => {
  const state = enqueueStreamingChunk(
    createStreamingDisplayState('```ts\n'),
    'const a = 1;\n',
  );
  assert.equal(state.displayed, '```ts\nconst a = 1;\n');
  assert.deepEqual(state.queue, []);
});

test('多行日志块级追加', () => {
  const chunk = 'stdout line 1\nstdout line 2\nstdout line 3\nstdout line 4\n';
  assert.equal(classifyStreamingChunk('', chunk), 'block');
});

test('tick 会释放队列并保留剩余内容', () => {
  const state = enqueueStreamingChunk(createStreamingDisplayState(), 'abcdef');
  const next = tickStreamingDisplay(state);
  assert.ok(next.displayed.length > 0);
  assert.ok(next.queue.length < 6);
});

test('done 后 flush 到真实完整内容', () => {
  const state = enqueueStreamingChunk(createStreamingDisplayState(), '你好');
  const flushed = flushStreamingDisplay(state, '你好，完整内容');
  assert.equal(flushed.displayed, '你好，完整内容');
  assert.deepEqual(flushed.queue, []);
});
```

- [x] **Step 3: 实现最小纯逻辑**

在 `streamingDisplay.ts` 中实现：

```ts
export type StreamingChunkMode = 'typewriter' | 'block';

export type StreamingDisplayState = {
  displayed: string;
  queue: string[];
};

export function createStreamingDisplayState(displayed = ''): StreamingDisplayState;
export function classifyStreamingChunk(currentDisplayed: string, chunk: string): StreamingChunkMode;
export function enqueueStreamingChunk(state: StreamingDisplayState, chunk: string): StreamingDisplayState;
export function tickStreamingDisplay(state: StreamingDisplayState): StreamingDisplayState;
export function flushStreamingDisplay(state: StreamingDisplayState, fullContent: string): StreamingDisplayState;
export function hasQueuedStreamingContent(state: StreamingDisplayState): boolean;
```

分类规则：

- `chunk.length > 240` 为 block。
- `chunk` 中换行数量大于等于 4 为 block。
- `currentDisplayed` 中未闭合 triple backtick 时为 block。
- `chunk.includes('```')` 为 block。
- 命中 `/(\[error\]|stdout|stderr|^\\s*[-+]\\s|^diff --git|^\\$\\s|^>\\s)/m` 为 block。
- 其他为 typewriter。

`tickStreamingDisplay` 根据队列长度动态释放：

- 队列小于 80：释放 1 个字符。
- 队列 80 到 240：释放 3 个字符。
- 队列大于 240：释放 8 个字符。

- 继续更新 React Query 完整 `content`。
- 对 `event.chunk` 非空调用 `appendChunk(event.messageId, event.chunk)`。
- `event.done` 时，从 React Query 中找到完整消息内容并调用 `finishMessage`。

- [x] **Step 3: 修改 ChatColumn/MessageBubble 参数**

- `ChatColumn` 接收 `getDisplayedContent` 与 `animatingMessageIds` 或等价 API。
- 渲染 `MessageBubble` 时传入 `displayContent`。
- `MessageBubble` 使用 `displayContent ?? message.content` 传给 `MessageContent`。
- `isStreaming` 应包含本地队列未清空状态，避免 done 后光标过早消失。
