# 任务协作 MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在现有聊天室中补齐任务协作 MVP：任务看板、任务创建、状态流转、Agent 指派、`/task` 快捷创建与基础 `@agent` 定向发送。

**Architecture:** 复用现有后端 `tasks` API、WebSocket 事件与 React Query 缓存；前端在 `RoomPage` 内引入右侧任务面板，并把任务操作封装为独立组件。第一轮只做房间内任务协作闭环，不实现甘特图、跨 Agent 自动编排、文件引用、代码 diff 或 OpenClaw 深度 session 生命周期。

**Tech Stack:** React 18、TypeScript、TanStack Query、Vite、TailwindCSS、Express、SQLite、WebSocket。

---

## Task 5: Composer `/task` 与基础 `@agent`

**Files:**
- Modify: `packages/frontend/src/pages/RoomPage.tsx`

- [ ] **Step 1: 实现 `/task` 快捷创建**

在 Composer 发送逻辑中识别：

```ts
const taskMatch = input.trim().match(/^\/task\s+(.+)/);
```

命中时调用 `api.createTask(roomId, { title: taskMatch[1] })`，成功后清空输入并刷新任务列表，不发送普通消息。

- [ ] **Step 2: 实现基础 `@agent` 定向发送**

从输入中解析 agent name：

```ts
const mentionNames = Array.from(input.matchAll(/@([\\w.-]+)/g)).map((m) => m[1]);
const mentions = agents
  .filter((agent) => mentionNames.includes(agent.agent_name) || mentionNames.includes(agent.agent_id))
  .map((agent) => agent.id);
```

调用：

```ts
api.sendMessage(roomId, input.trim(), mentions.length > 0 ? mentions : undefined);
```

- [ ] **Step 3: 更新 placeholder**

当存在 Agent 时 placeholder 改为：

