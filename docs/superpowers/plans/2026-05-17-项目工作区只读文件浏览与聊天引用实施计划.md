# 项目工作区只读文件浏览与聊天引用 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 OpenDeepSea 增加受保护的项目工作区只读文件浏览能力，并允许在聊天消息框中快捷引用真实项目文件。

**Architecture:** 后端以 `projects.path` 为 workspace root，新增受 token 和可信 Origin 保护的只读 workspace API，并用共享路径解析模块统一处理 realpath、symlink、忽略规则、搜索限制和文件元信息。消息发送新增 `workspaceFilePaths`，workspace 引用只写入 message metadata；dispatcher 按项目路径解析 workspace `localPath` 注入 agent prompt。前端新增 workspace 文件选择器、composer 第三类附件、metadata union 解析和 workspace 引用卡片。

**Tech Stack:** Node.js、Express、SQLite、TypeScript、React 18、Vite、TanStack Query、lucide-react、node:test。

**Token 方案：**

首版采用唯一明确方案：`OPENDEEPSEA_LOCAL_TOKEN` 环境变量。

- 后端启动时读取 `process.env.OPENDEEPSEA_LOCAL_TOKEN`。
- 如果未设置，后端启动时生成随机 token，并在日志中打印一次：`[server] local access token: <token>`。
- `local-access.ts` 暴露 `getLocalAccessToken()`，作为 workspace API guard 的唯一 token 源。
- Vite dev proxy 从 `process.env.OPENDEEPSEA_LOCAL_TOKEN` 读取同一 token，并给 `/api/projects/*/workspace/*` 请求注入 `X-OpenDeepSea-Local-Token`。
- 非 proxy 场景下，`api.ts` 允许从 `localStorage.getItem('opendeepsea.localToken')` 读取 token 并添加到 workspace API 请求头；不要求普通 API 携带 token。
- 任何实现者不得为了前端可用性放宽 workspace API token 校验。
