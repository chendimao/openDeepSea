# 纯 ACP 多智能体聊天室 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将房间主路径改造成纯 ACP 多智能体聊天室，去掉自动任务化和工作流化，公开显示各智能体原始回复、完整 thinking 与工具轨迹，并由 planner 以“建议后暂停”为默认模式进行下一步调度。

**Architecture:** 保留现有 `dispatchUserMessage -> respondAsAgent -> agent_run/message/ws` 流式底座，移除主路径上的 `task_readiness / collaboration_decision / workflow` 升级链，新增 `planner_decision` 与结构化 `trace`。前端 `RoomPage` 收敛为统一消息时间线，只保留 planner 决策区、thinking 折叠区和工具轨迹折叠区。

**Tech Stack:** TypeScript, Node.js, Express, better-sqlite3, React 18, Vite, React Query, WebSocket
