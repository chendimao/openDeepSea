# 项目与群聊筛选搜索 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为项目列表增加项目名称筛选，并为项目内群聊增加覆盖名称、描述、消息和任务的语义搜索，模型不可用时自动降级关键词搜索。

**Architecture:** 项目列表筛选完全在前端完成；群聊搜索由后端提供项目级搜索接口，后端聚合 rooms、messages、tasks 候选内容，优先调用现有模型入口做 LLM rerank，失败时走本地关键词搜索。前端通过 React Query 按输入状态在普通群聊列表和搜索结果之间切换，并显示搜索数量与降级提示。

**Tech Stack:** TypeScript、Node.js、Express、better-sqlite3、node:test、React 18、Vite、TanStack Query、Tailwind utility classes、现有 `invokeConfiguredModelText` 模型调用。

```ts
import { invokeConfiguredModelText } from './chat-model.js';
import { db } from './db.js';
import { projectRepo } from './repos/projects.js';
import type { Room, RoomSearchMatchedField, RoomSearchResponse, RoomSearchResult, Task, Message } from './types.js';

export interface SearchProjectRoomsInput {
  projectId: string;
  query: string;
  invokeModel?: typeof invokeConfiguredModelText;
  forceKeywordOnly?: boolean;
}

interface RoomCandidate {
  room: Room;
  messages: Message[];
  tasks: Task[];
}

const STOP_WORDS = new Set([
  '的',
  '了',
  '和',
  '与',
  '或',
  '群聊',
  '搜索',
  '找到',
  '找出',
]);

export async function searchProjectRooms(input: SearchProjectRoomsInput): Promise<RoomSearchResponse> {
  const query = input.query.trim();
  if (!query) {
    throw new Error('query is required');
  }
  if (!projectRepo.get(input.projectId)) {
    throw new Error('project not found');
  }

  const candidates = listRoomCandidates(input.projectId);
  const keywordResults = keywordSearch(candidates, query);

  if (input.forceKeywordOnly) {
    return buildResponse(query, 'keyword', false, null, keywordResults);
  }

  // Task 2 会补充模型 rerank。
  return buildResponse(query, 'keyword', false, null, keywordResults);
}
```

实现辅助函数：

- `listRoomCandidates(projectId)`：
  - `SELECT * FROM rooms WHERE project_id = ? ORDER BY created_at DESC`
  - `SELECT messages.* FROM messages JOIN rooms ON rooms.id = messages.room_id WHERE rooms.project_id = ? ORDER BY messages.created_at DESC`
  - `SELECT * FROM tasks WHERE project_id = ? ORDER BY created_at DESC`
  - 按 `room_id` 分组。
- `extractSearchTerms(query)`：
  - 小写归一化。
  - 英文/数字按 token 提取。
  - 中文先保留完整 query，再用停用词切分出长度大于 1 的片段。
  - 对示例 query 至少能得到 `页面`、`显示`、`不完整`、`bug` 中的有效项。
- `keywordSearch(candidates, query)`：
  - 对 room name、description、message content、task title、task description 逐项匹配。
  - 命中加权：name 50、description 35、task title 30、task description 20、message 15。
  - 返回 score 大于 0 的结果，按 score 降序，再按 room.created_at 降序。
  - `highlights` 最多 3 条，文本截断到合理长度。

在 `room-search.ts` 增加：

```ts
interface ModelRoomSearchPayload {
  query: string;
  rooms: Array<{
    id: string;
    name: string;
    description: string | null;
    messages: string[];
    tasks: Array<{ title: string; description: string | null }>;
  }>;
}

interface ParsedModelResult {
  roomId: string;
  score: number;
  reason: string;
}
```

实现：

- `buildModelPayload(query, candidates)`：
  - 每个 room 最多 8 条消息、8 个任务。
  - 单条文本截断到 240 字符。
  - 单个 room 候选总量保持小而可读。
- `buildModelMessages(payload)`：
  - 使用 `SystemMessage` 和 `HumanMessage`。
  - 系统消息要求只输出 JSON，不要解释。
  - 明确 score 范围 0 到 1，低于 0.2 的不返回。
- `parseModelResults(raw, roomById)`：
  - `JSON.parse`。
  - 只接受 `results` 数组。
  - 忽略未知 room id。
  - score clamp 到 0 到 1。
  - 过滤 score < 0.2。
  - 根据模型顺序或 score 降序返回。

```ts
router.get('/projects/:projectId/rooms/search', async (req, res) => {
  const schema = z.object({
    q: z.string().trim().min(1),
  });
  const parsed = schema.safeParse(req.query);
  if (!parsed.success) return res.status(400).json({ error: parsed.error.flatten() });
  try {
    res.json(await searchProjectRooms({
      projectId: req.params.projectId,
      query: parsed.data.q,
    }));
  } catch (err) {
    const message = (err as Error).message;
    if (message === 'project not found') return res.status(404).json({ error: 'project not found' });
    if (message === 'query is required') return res.status(400).json({ error: 'query is required' });
    res.status(500).json({ error: 'room search failed' });
  }
});
```

## 风险控制

- 若 LLM 输出不稳定，优先保证关键词降级路径可靠。
- 若候选文本太长导致模型慢，先收紧每个 room 的消息和任务数量，不引入新索引。
- 若前端布局在移动端拥挤，优先让搜索框独占一行，保留“新建群聊”按钮可点击。
- 若全量后端测试暴露旧测试不稳定，先记录失败项和日志，不把未验证状态描述为通过。
