# 去除 OpenClaw 依赖与任务流转验收 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 移除 OpenClaw 运行时和用户可见依赖，完成 ACP-only + LangGraph 任务流转、自动记忆和聊天页视口布局的真实浏览器验收。

**Architecture:** 先删除 OpenClaw Gateway 功能入口，再把模型配置抽为记忆蒸馏的唯一 LLM 来源。任务命令统一经后端消息创建路径处理，按钮操作走带聊天回写的 room-scoped API；LangGraph 启动和审批都改为短请求 + 后台推进。前端只保留 ACP agent 入口，并为真实浏览器测试提供稳定定位点和可视区域滚动约束。

**Tech Stack:** TypeScript, Node.js, Express, SQLite, LangGraph, LangChain OpenAI-compatible API, React 18, Vite, Playwright/browser smoke, ACP CLIs.

---

## Scope Check

本计划覆盖多个模块，但它们服务于同一条验收主线：去 OpenClaw 后，用户能在群聊中用 ACP agent、任务和 LangGraph 完整流转。执行时按任务顺序推进；共享类型、路由、数据库兼容和根依赖禁止并行写入。前端布局和浏览器验收可在后端主线稳定后并行收尾。
