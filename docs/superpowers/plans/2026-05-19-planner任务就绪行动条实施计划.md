# Planner 任务就绪行动条 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 当 planner 已经整理出可执行任务信息时，在消息气泡中提示用户选择“开始任务”或“继续沟通”。

**Architecture:** 后端继续保持用户确认后才创建任务和启动 workflow；前端通过结构化 `message.metadata.task_readiness` 判断是否展示行动条。第一版不让 planner 自动越过用户确认，也不引入新表。

**Tech Stack:** TypeScript, React 18, Express, SQLite, node:test, Vite.

### Task 2: Planner 气泡行动条

**Files:**

- [x] **Step 1: 复用现有 promote API**

`开始任务` 优先调用 `api.promoteMessageToWorkflow(roomId, sourceMessageId)`；如果 metadata 没有 `source_message_id`，使用当前 planner 消息 id。

- [x] **Step 2: 添加气泡行动条**

在 planner/agent 消息正文下方展示：

```text
已具备创建任务的基础信息
[开始任务] [继续沟通]
```

`继续沟通` 只聚焦输入区或保持无副作用关闭提示，第一版不写后端状态。
