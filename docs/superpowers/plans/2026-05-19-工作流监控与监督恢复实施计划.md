# 工作流监控与监督恢复 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 OpenDeepSea 工作流增加 incident-driven 的运行监控、planner/supervisor 恢复决策、策略化恢复执行和聊天室透明化反馈，覆盖 v1 基础恢复与 v2 拆分/换人能力。

**Architecture:** 使用 deterministic workflow monitor 发现事实异常，落库为 `workflow_incidents`，再由 recovery supervisor 结合默认策略或 LLM planner 生成结构化恢复动作，最后由 recovery executor 幂等执行 retry/reassign/split/ask_user/mark_blocked。planner 不做轮询心跳，只在 incident 出现后参与决策；无 LLM 配置时默认策略仍可运行。

**Tech Stack:** Node.js、TypeScript、Express 后端、SQLite/better-sqlite3、LangChain ChatOpenAI、现有 LangGraph workflow runtime、node:test。

## 核心数据模型

新增 incident 类型：

```ts
export type WorkflowIncidentType =
  | 'backend_restart_interrupted'
  | 'agent_run_stale'
  | 'step_without_active_run'
  | 'child_task_failed'
  | 'executor_unavailable'
  | 'runtime_boundary_mismatch'
  | 'planner_output_invalid'
  | 'unknown';

export type WorkflowIncidentStatus =
  | 'open'
  | 'deciding'
  | 'executing'
  | 'resolved'
  | 'blocked'
  | 'ignored';

export type WorkflowRecoveryAction =
  | 'retry_same_agent'
  | 'retry_with_global_agent'
  | 'reassign_agent'
  | 'split_task'
  | 'ask_user'
  | 'mark_blocked';
```

`workflow_incidents` 表字段：

```sql
id TEXT PRIMARY KEY,
room_id TEXT NOT NULL,
project_id TEXT NOT NULL,
workflow_run_id TEXT NOT NULL,
workflow_step_id TEXT,
task_id TEXT NOT NULL,
child_task_id TEXT,
agent_run_id TEXT,
room_agent_id TEXT,
incident_type TEXT NOT NULL,
status TEXT NOT NULL,
severity TEXT NOT NULL DEFAULT 'warning',
fingerprint TEXT NOT NULL,
error TEXT,
context_json TEXT NOT NULL DEFAULT '{}',
decision_json TEXT,
action TEXT,
action_status TEXT,
attempt_count INTEGER NOT NULL DEFAULT 0,
last_message_id TEXT,
created_at INTEGER NOT NULL,
updated_at INTEGER NOT NULL,
resolved_at INTEGER,
UNIQUE(workflow_run_id, fingerprint)
```

fingerprint 规则：

```ts
fingerprint = hash([
  workflow_run_id,
  workflow_step_id ?? '',
  child_task_id ?? task_id,
  incident_type,
  normalizeError(error),
].join('|'))
```

## 风险与验收

**风险：**
- 自动恢复可能重复启动 agent run：通过 incident fingerprint、active run guard、resolved status 防止。
- LLM 输出不稳定：通过 zod parser 和 default policy 兜底。
- backend restart 后状态混乱：通过 startup monitor 将 orphan 状态转为 incident，再走统一恢复路径。
- 同一子任务无限重试：通过 attempt_count 和同 child incident history 上限升级到 ask_user/mark_blocked。
- `types.ts` 当前已有资源资产改动：实施时只追加本功能类型，避免覆盖用户改动。

**验收标准：**
- backend restart/interrupted 不再静默卡死；会产生 incident 和聊天室决策说明。
- stale agent run、running step without active run、failed child task 都能被 monitor 检测。
- planner/supervisor 能输出并执行 6 类动作：retry_same_agent、retry_with_global_agent、reassign_agent、split_task、ask_user、mark_blocked。
- 同一子任务连续中断 2 次后不再盲目 retry。
- 所有恢复动作都有聊天室可见消息。
- 相关 backend tests 和 build 通过。
