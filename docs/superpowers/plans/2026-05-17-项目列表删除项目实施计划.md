# 项目列表删除项目 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 Dashboard 项目列表增加低误触删除入口，并让后端删除项目时保护运行中任务、清理内部孤儿设置且不触碰真实项目目录。

**Architecture:** 后端把项目删除收敛到 `projectRepo.delete()` 的受保护事务，路由负责把删除结果映射为 HTTP 状态。前端在项目卡片右上角增加 Radix DropdownMenu 入口和确认 Dialog，复用现有 `api.deleteProject(id)`、React Query 和 toast。

**Tech Stack:** Node.js、Express、better-sqlite3、node:test、React 18、TypeScript、Vite、TanStack Query、Radix UI、lucide-react、sonner。

- [x] **Step 3: 实现 `projectRepo.delete()` 结构化结果**

在 `packages/backend/src/repos/projects.ts` 增加类型：

```ts
type DeleteProjectResult =
  | { ok: true }
  | { ok: false; reason: 'not_found' }
  | { ok: false; reason: 'active_runs'; activeAgentRunCount: number; activeWorkflowRunCount: number };
```

实现逻辑：

- 项目不存在返回 `not_found`。
- 查询项目内 active agent runs：
  - `agent_runs.status IN ('running', 'queued')`
  - 通过 `agent_runs.room_id -> rooms.project_id` 关联项目。
- 查询项目内 active workflows：
  - `workflow_runs.status IN ('draft', 'running', 'awaiting_decision', 'awaiting_approval', 'blocked')`
  - `workflow_runs.project_id = ?`
- 有活动项返回 `active_runs`。
- 无活动项时使用 `db.transaction()`：
  - 删除项目下 room scope settings：
    `DELETE FROM settings WHERE scope = 'room' AND scope_id IN (SELECT id FROM rooms WHERE project_id = ?)`
  - 删除 project scope settings：
    `DELETE FROM settings WHERE scope = 'project' AND scope_id = ?`
  - 删除 `projects` 行。

- [x] **Step 4: 更新路由 HTTP 映射**

在 `packages/backend/src/routes.ts`：

```ts
const result = projectRepo.delete(req.params.id);
if (result.ok) return res.status(204).end();
if (result.reason === 'not_found') return res.status(404).json({ error: 'not found' });
return res.status(409).json({
  error: 'project has active runs',
  active_agent_run_count: result.activeAgentRunCount,
  active_workflow_run_count: result.activeWorkflowRunCount,
});
```
