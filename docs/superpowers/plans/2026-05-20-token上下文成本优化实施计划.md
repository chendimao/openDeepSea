# Token 上下文成本优化 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 OpenDeepSea 的 agent 调用默认更省 token，并把每次运行的上下文成本可视化、可追踪、可调优。

**Architecture:** 先在 `agent_runs` 增加上下文预算 ledger，记录 prompt 字符数、估算 token、记忆条数和 session resume 状态；再引入统一 Context Packer，将普通群聊、协作/工作流 prompt 的记忆、上游输出和 session 策略集中装配。默认策略改为短会话优先、置顶记忆少量必带、其余记忆按当前请求相关性召回，工作流/多 agent 通过结构化摘要和 blackboard 传递状态而不是重复传全文。

**Tech Stack:** TypeScript, Node.js, Express, SQLite, React 18, Vite, node:test.

默认 profile：

- `forceResumeSessionId` 存在时使用它。
- `profile === 'full' && allowSavedSessionResume` 时使用 `agentSessionId`。
- 其他情况使用 `null`。

## 风险与回退

- **ACP session 行为变化:** 默认短会话会降低连续对话能力。回退方式是把 profile 调为 `full` 或为特定 agent 开启 `allowSavedSessionResume`。
- **相关召回漏记忆:** 第一版关键词召回可能漏掉语义相关记忆。先保留 pinned 必带和 full profile，后续再接 embedding/RAG。
- **估算 token 不等于真实计费:** 当前用字符估算只做趋势观测。后续若 ACP backend 暴露 usage，再补 `actual_input_tokens` / `actual_output_tokens`。
- **workflow prompt 过度压缩:** blackboard 必须保留 artifact/message refs，确保 agent 需要全文时可追溯拉取。
- **DB migration 风险:** 新列只增不删，默认值保证旧数据可读；上线前用本地真实 DB 跑一次启动验证。

## 验收标准

- 普通群聊 agent run 默认 `session_reused = 0`，adapter 收到 `sessionId = null`。
- `agent_runs` 每条新记录都有 prompt 字符数、估算 token、memory 条数、memory 字符数、profile 和 ledger JSON。
- 前端 run 面板能看到估算 token、memory 条数和 fresh/resume 状态。
- `lean / normal / full` 三档策略可被测试覆盖。
- workflow 多 agent 阶段通过 blackboard 摘要传递状态，不再把上游长输出全文重复塞进 prompt。
- `npm run build` 通过。
