# 任务意图结构化分流 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为群聊中的 planner 输出增加结构化任务意图分流，避免“只做分析/方案/文档、不进入实现”的内容被错误送入默认开发闭环。

**Architecture:** 在消息 metadata 中新增 `execution_intent`，由后端 readiness 推断、前端入口、workflow supervisor 和内置 workflow 共同消费。实现类任务继续走默认开发闭环；只读/方案/文档类任务走轻量方案闭环或停留在群聊，不再触发代码执行和代码审查。

**Tech Stack:** Node.js、Express、SQLite、TypeScript、React 18、Vite、node:test、现有 workflow definitions / LangGraph runtime。

---

## 任务边界

- 本计划不实现复杂临时 workflow 生成。
- 本计划不改数据库 schema；`execution_intent` 放在既有 JSON metadata、task description 或 workflow snapshot audit 中。
- 本计划不删除默认开发闭环。
- 本计划不改变用户显式通过 `/task` 或任务按钮创建实现任务的能力。
- 本计划不引入新的模型依赖。

- [ ] **Step 4: 实现 intent 判定**

在 `dispatcher.ts` 新增内部类型和 helpers：

```ts
type InferredTaskExecutionIntent =
  | 'analysis_only'
  | 'planning_only'
  | 'documentation_only'
  | 'implementation'
  | 'debug_fix'
  | 'review_only';

function inferTaskExecutionIntent(text: string): InferredTaskExecutionIntent | null {
  if (matchesAny(text, [
    /不进入实现/,
    /不修改代码/,
    /不改文件/,
    /只做方案/,
    /只做.*规则/,
    /只读分析/,
    /本轮不要求代码实现/,
    /未进入实现/,
  ])) return 'analysis_only';

  if (matchesAny(text, [/只做文档/, /文档说明/, /产品规则说明/])) return 'documentation_only';
  if (matchesAny(text, [/只做代码审查/, /review only/i, /审查.*不修改/])) return 'review_only';
  if (matchesAny(text, [/修复/, /bug/, /故障/, /报错/, /阻塞/]) && matchesAny(text, [/实现/, /修改/, /改动/, /提交/])) return 'debug_fix';
  if (matchesAny(text, [/实施目标/, /实施范围/, /下一步可以进入工程排期/, /修改代码/, /实现/])) return 'implementation';
  return null;
}

function matchesAny(text: string, patterns: RegExp[]): boolean {
  return patterns.some((pattern) => pattern.test(text));
}
```

调整 `inferTaskReadiness`：

```ts
const executionIntent = inferTaskExecutionIntent(text);
if (!executionIntent) return null;
...
recommended_mode: isImplementationIntent(executionIntent) ? 'formal_workflow' : 'chat_collaboration',
execution_intent: executionIntent,
```

其中：

```ts
function isImplementationIntent(intent: InferredTaskExecutionIntent): boolean {
  return intent === 'implementation' || intent === 'debug_fix';
}
```

## 风险与回滚

- 风险：放宽 workflow definition shape 可能影响用户自定义 workflow 校验。
  - 控制：仅允许两种明确 shape：开发闭环 shape、方案文档 shape。
- 风险：旧消息没有 `execution_intent`。
  - 控制：前端 `undefined` 保持旧行为；后端 workflow fallback 对缺失 intent 仍使用默认开发闭环。
- 风险：模型 supervisor 仍可能选错。
  - 控制：runtime deterministic fallback 和 system prompt 双保险；低置信不采用模型选择。
- 风险：只读方案任务仍被用户强行开始。
  - 控制：promote-to-workflow 持久化 intent；workflow selection 根据 intent 选择方案闭环。

## 完成标准

- `TaskReadinessMetadata` 支持 `execution_intent`。
- 含“不进入实现/只做方案/只读分析”的 planner 输出不会推荐默认开发闭环。
- 前端不再把只读方案输出展示成“开始任务”的主入口。
- 内置 workflow 列表包含“默认开发闭环”和“方案文档闭环”。
- workflow supervisor 和 fallback 都能基于 intent 选择 workflow。
- 方案闭环不执行代码、不走代码审查、不要求 build/commit。
- 现有实现类任务仍走默认开发闭环。
- 相关测试与 `npm run build` 通过。
