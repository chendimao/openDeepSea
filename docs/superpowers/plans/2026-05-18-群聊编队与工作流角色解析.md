# 群聊编队与工作流角色解析 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 创建群聊时可选择协作编队，工作流按角色和能力选择可执行智能体，并为高级用户保留步骤覆盖能力。

**Architecture:** 新增 Room Crew Template 作为房间初始化层，复用现有内置 agent templates。新增可测试的 Workflow Role Resolver，把 `workflow_role`、`capabilities`、ACP 可执行状态和任务 scope 合并成选择规则。Step Override 复用已有 `assigned_agent_id` 与 WorkflowBuilder 节点 role/agent 字段，先补运行时 resolver 支撑。

**Tech Stack:** Node.js、Express、SQLite、TypeScript、React 18、Vite、TanStack Query。
