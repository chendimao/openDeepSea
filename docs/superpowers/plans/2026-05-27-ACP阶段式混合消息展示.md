# ACP 阶段式混合消息展示 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 agent 回复从“完整正文 + 完整 ACP timeline”改为“调查 / 修改 / 验证 / 总结”阶段式混合展示，每个阶段展示对应正文摘要和相关 ACP 事件。

**Architecture:** 新增前端阶段模型，复用已有 `AgentTimeline` 渲染事件，不改 ACP provider、后端协议和数据库 schema。`MessageContent` 在有 trace 且为 preview 模式时使用阶段式视图；source 模式和无 trace 消息保持原有渲染。

**Tech Stack:** React 18、TypeScript、Vite、Node test runner、现有 CSS。

---

### Task 1: 阶段模型

**Files:**
- Create: `packages/frontend/src/components/agent-timeline/phases.ts`
- Test: `packages/frontend/src/components/agent-timeline/phases.test.ts`

- [x] **Step 1: 定义阶段类型**

  阶段固定为 `investigation`、`changes`、`verification`、`summary`，每个阶段包含 `title`、`body`、`events`。

- [x] **Step 2: 事件分组规则**

  `Search/Read/List/View` 等工具事件归入调查阶段；`file_diff`、写入或 patch 类工具归入修改阶段；测试、build、lint、typecheck 命令归入验证阶段；剩余事件归入调查阶段。

- [x] **Step 3: 正文切分规则**

  优先识别 Markdown 标题或加粗标签中的“调查/分析/修改/实现/验证/测试/总结/结论”；识别不到时保留完整正文到总结阶段，避免误切。
