# 去除轻量群聊协作只保留普通群聊与 workflow Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 删除轻量群聊协作模式，保留普通群聊和正式 workflow 两条清晰路径。

**Architecture:** 后端停止生成和执行 `collaboration_decision`，移除 `/collaborations` 轻量协作接口和 runner；前端删除协作选择卡片与启动轻量协作入口。`promote-to-workflow` 继续作为从群聊消息启动正式 workflow 的唯一正式执行入口，并保留必要历史 metadata 兼容。

**Tech Stack:** TypeScript, Express, SQLite, React 18, TanStack Query, Vite, node:test。

---

## 执行前约束

- 当前工作区已有大量未提交修改。实施前先运行 `git status --short`，确认本任务外修改不要被误提交。
- 本任务不适合并行写入：涉及 shared types、routes、dispatcher、前端 metadata 和 RoomPage，串行执行更稳。
- 不做破坏性数据库迁移。第一轮只停止轻量协作的新写入和公开入口，保留历史 SQLite 字段以避免本地数据损坏。
